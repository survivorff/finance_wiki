# 3.7 高可用与安全工程

> 深度：🔴 硬核
> 7×24 不能停、管着真金白银的系统，对可用性和安全的要求是「偏执级」的。

## 一、高可用（High Availability）

### 为什么 CEX 对可用性极度敏感
- **7×24 不停市**：加密市场没有「收盘」，任何时候宕机都有用户在交易。
- **宕机 = 直接损失**：行情剧烈时宕机，用户无法平仓/止损，会造成真实资金损失和大量索赔、声誉崩塌。
- ⚠️ 「**该跌的时候登不上、平不了仓**」是用户对交易所最大的怨念之一，也常引发「故意宕机」的阴谋论。

### 高可用手段

| 手段 | 说明 |
| --- | --- |
| **多活/多机房** | 服务跨多个数据中心/可用区部署，单点故障不影响整体 |
| **撮合主备热切换** | 状态机复制（见 [3.2](./3.2-撮合引擎.md)），主挂备秒级接管 |
| **无状态服务水平扩展** | 网关、行情等无状态服务可弹性扩容应对流量峰值 |
| **限流降级** | 流量洪峰时保护核心链路（保撮合、保提现），降级非核心功能 |
| **熔断隔离** | 某服务故障时熔断，避免雪崩拖垮全站 |

### 极端行情下的「容量地狱」
- 重大行情（暴涨暴跌）时，流量可能瞬间涨 10-100 倍：下单、撤单、查询、行情订阅全部爆炸。
- 这时系统最容易崩。所以要做**压测、容量预留、自动扩容、限流降级预案**。
- 💡 行业潜规则：很多所宁可「限流卡顿」也要保住撮合和资金安全不出错——卡顿可以骂，资金错了是灭顶之灾。

## 二、容灾与数据可靠

- **多副本 + 异地备份**：核心数据（账户、订单、日志）多副本存储，异地容灾。
- **事件溯源做最终真相**：所有交易指令持久化（见 [3.2](./3.2-撮合引擎.md)），可重放重建状态。
- **定期演练**：故障切换、数据恢复要定期实战演练（不演练的预案=没有预案）。

## 三、安全工程（攻防）

交易所是「全世界黑客最想攻破的金库之一」，安全是全方位的。

### 安全层次

```
┌─ 资产安全     冷热钱包、多签/MPC、签名验证、提币风控 (见 3.3)
├─ 应用安全     防 SQL注入/XSS/越权、代码审计、依赖扫描
├─ 网络安全     DDoS 防护、WAF、流量清洗
├─ 账户安全     2FA、防撞库、设备指纹、风控
├─ 内部安全     权限最小化、操作审计、防内鬼、敏感操作多人复核
└─ 供应链安全   第三方组件/依赖/服务商的安全(Bybit 教训)
```

### 重点攻击面与防御

| 攻击面 | 防御 |
| --- | --- |
| **热钱包私钥泄露** | 冷热分离、MPC、最小化热钱包余额 |
| **签名流程被欺骗** | 签名前严格验证、防盲签、白名单（Bybit 教训） |
| **DDoS** | 流量清洗、CDN、弹性扩容 |
| **撞库/盗号** | 强制 2FA、异常登录风控、提币冷静期 |
| **内部作恶** | 权限分离、操作审计、多人复核、最小权限 |
| **API 密钥泄露** | 权限分级（只读/交易/提币分开）、IP 白名单 |
| **供应链攻击** | 审查第三方依赖和服务商 |

### 内部风险（最易被忽视）💡
- **内鬼**：有权限的员工监守自盗或配合外部。
- 防御：**职责分离**（没有任何单人能独自动用资金）、敏感操作多人审批、全程审计日志。
- 💡 多签/MPC 不只是防外部黑客，也是**防内部单点作恶**——这就是为什么动用冷钱包要多人签名。

## 四、性能工程小结

把前面各篇的性能手段汇总：

| 子系统 | 关键性能手段 |
| --- | --- |
| 撮合 | 单线程、全内存、绑核、内存预分配、无锁队列 |
| 账户 | 预冻结、乐观锁、账户分片、串行化 |
| 行情 | WebSocket、增量推送、多级缓存、边缘分发 |
| 整体 | 异步化、批处理、消息队列削峰、读写分离 |

## 五、可观测性（Observability）

- **监控告警**：核心指标（撮合延迟、订单积压、对账差异、提币队列）实时监控，异常即告警。
- **链路追踪**：一笔订单跨多个服务，要能全链路追踪定位问题。
- **对账监控**：账面 vs 链上的差异是「头等告警」，可能意味着 bug 或被盗。

## 六、灰度与发布

- **金融系统改代码如履薄冰**：核心交易链路的变更要灰度、可回滚、充分测试。
- **影子测试/回放**：用真实历史订单流回放验证新版本撮合结果一致。
- 💡 撮合引擎是确定性状态机的好处之一：**可以用历史订单流回放来验证新代码不改变行为**。

## 小结

- 7×24 不停市 + 管钱，让 CEX 对可用性和安全有「偏执级」要求。
- 高可用靠多活、撮合主备热切换、限流降级、容量预留（极端行情是大考）。
- 安全是全方位的：资产、应用、网络、账户、内部、供应链。
- 内部作恶用职责分离 + 多人复核 + 多签防范。
- 可观测性（尤其对账监控）和灰度回放是金融系统的护身符。
