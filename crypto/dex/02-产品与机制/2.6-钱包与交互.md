# 2.6 钱包与交互

> 深度：🟢 入门 → 🟡
> DEX 没有「注册账户」，钱包就是你的身份和资产。理解钱包交互，是用 DEX 的第一课，也是安全的第一课。

## 钱包 = 你的链上身份 + 资产 + 签名工具

```
钱包做三件事:
1. 保管私钥 (= 你的资产控制权)
2. 你的地址 (= 你的链上身份, 公开可查)
3. 对交易签名 (= 授权"用我的资产做某事")
```

- **没有用户名密码、没有注册**：你的钱包地址就是账户，私钥就是唯一凭证。
- **私钥/助记词 = 一切**：谁有它谁就拥有里面的资产。丢了 = 永久失去，泄露 = 立即被盗，**无人能找回**。

### 钱包类型
| 类型 | 例子 | 特点 |
| --- | --- | --- |
| 浏览器插件钱包 | MetaMask, Phantom, Rabby | 最常用，连接 DEX 网站 |
| 移动端钱包 | Trust Wallet, OKX Wallet | 手机用 |
| 硬件钱包 | Ledger, Trezor | 私钥离线，最安全 |
| 智能合约钱包 | Safe(多签), AA 钱包 | 多签、社交恢复、账户抽象 |

## 一笔 DEX 交易的完整流程 ⭐

以「在 Uniswap 用 USDC 换 ETH」为例：

```
1. 连接钱包 (connect wallet) — 网站读取你的地址(只读, 不动钱)
2. 选择交易对、输入数量 — 前端计算预计获得 + 滑点
3. [首次] 授权 approve — 允许 Uniswap 合约动用你的 USDC(见下文)
4. 确认交易 swap — 钱包弹窗显示细节, 你签名
5. 交易广播上链 → 等待打包确认
6. 成交 — ETH 进你钱包, 链上可查
   全程 USDC/ETH 没离开过你的控制(自托管)
```

## ⚠️ 授权（Approve）：DEX 最大的安全陷阱 ⭐

这是 DEX 特有、也是最多人栽跟头的地方。

### 为什么需要 approve
```
ERC20 代币的机制: 合约不能直接动你的币, 你要先"授权(approve)"它能动多少
  → 用 DEX 换 USDC 前, 要先授权 DEX 合约"可以动用我的 USDC"
```

### 陷阱：无限授权（Unlimited Approval）⚠️
```
很多 DApp 默认请求"无限授权"(approve 一个极大值)
  → 方便: 以后交易不用每次都 approve
  → 危险: 如果这个合约有漏洞 或 是恶意的, 它能把你授权的币【全部转走】!
```

> ⚠️ **真实损失场景**：用户签了某个恶意网站的「无限授权」，几个月后钱包里的币突然被盗刷——
> 因为那个恶意合约一直有权动你的币。**钓鱼网站最常用的就是骗你签授权。**

### 防护
```
1. 只授权需要的额度(而非无限), 或用支持"按需授权"的钱包
2. 定期检查并撤销授权 (revoke.cash 等工具)
3. 用 Rabby 等会模拟交易、提示风险的钱包
4. 绝不在不明网站连接钱包/签授权
```

## ⚠️ 签名钓鱼（最危险）

除了授权，还有更隐蔽的**签名钓鱼**：

```
恶意网站让你签一个看似无害的"消息签名"(免 Gas, 没弹转账)
  实际是 Permit / Permit2 授权 或 一个转移资产的离线订单
  → 你以为只是"登录签名", 其实签走了资产授权
```

- ⚠️ **Permit 类签名**：链下签名即可授权，钓鱼者最爱（你没花 Gas，放松警惕）。
- **铁律**：看不懂的签名**绝不签**。正规操作不会让你签看不懂的东西。

## DEX 交互的其他坑

| 坑 | 说明 | 防护 |
| --- | --- | --- |
| **转错链/地址** | 不可逆，钱没了 | 仔细核对，先小额测试 |
| **假代币/仿冒** | 同名不同合约地址 | 用官方合约地址，别信搜索结果 |
| **貔貅盘** | 能买不能卖的骗局币 | 见 [06 安全](../06-风险与安全/README.md) |
| **滑点设置** | 太高被夹，太低交易失败 | 主流币用默认，谨慎调高 |
| **Gas 不足** | 交易卡住/失败 | 留足原生币(ETH/SOL)付 Gas |
| **假网站/钓鱼** | 域名仿冒 | 用书签，核对 URL |

## 自托管的「自由与责任」⭐

> DEX 把资产控制权完全交给你，代价是**全部责任也在你**：
> - 没有「忘记密码」找回——私钥丢了就是丢了。
> - 没有客服撤销——转错、签错就是错了。
> - 没有平台兜底——踩坑血本无归。
>
> 这是去中心化的本质：**最大的自由 = 最大的自我负责。** 用 DEX 的第一课，是学会敬畏「不可逆」和「签名」。

## 小结

- 钱包是链上身份+资产+签名工具，私钥/助记词即一切，丢失泄露均无法挽回。
- DEX 交易流程：连钱包→输入→授权(approve)→签名 swap→上链确认，全程自托管。
- 最大陷阱是「无限授权」和「签名钓鱼」（尤其 Permit 类链下签名）——看不懂的绝不签，定期撤销授权。
- 转错地址、假币、貔貅、钓鱼网站都是高发坑，且不可逆、无兜底。
- 自托管=最大自由+最大责任，敬畏「不可逆」是第一课。
