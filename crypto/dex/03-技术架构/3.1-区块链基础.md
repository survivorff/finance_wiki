# 3.1 区块链基础回顾（DEX 的底座）

> 深度：🟢 概念入门 → 🔴 关键细节
> DEX 跑在区块链上，理解几个底层概念是读懂后面所有内容的前提。已熟悉的可跳过。

## 为什么要先讲这个

DEX 的一切特性（自托管、不可逆、Gas、慢、MEV）都源于区块链的底层机制。不理解底座，就理解不了 DEX 为什么是现在这样。

## 一、账户与地址

```
账户 = 一对密钥:
  私钥(private key): 你独有的秘密, 用来签名(= 资产控制权), 绝不能泄露
  公钥/地址(address): 公开的"账号", 别人往这里转账, 链上可查

助记词(seed phrase): 12/24 个单词, 能恢复出私钥(所以等同于私钥, 同样要命)
```

- ⚠️ 这就是自托管的根：**控制权 = 私钥**。没有公司帮你管，丢了没人能恢复。

### 两种账户（以太坊）
- **EOA（外部账户）**：普通用户钱包，由私钥控制。
- **合约账户**：智能合约，由代码控制（DEX 就是这种）。

## 二、交易与 Gas ⭐

```
交易(transaction): 你签名的一条指令(转账/调用合约), 广播到网络等打包
Gas: 执行交易要消耗的"计算燃料", 用原生币(ETH/SOL等)支付给矿工/验证者
     Gas 费 = Gas 用量 × Gas 单价
     越复杂的操作(如多跳 swap)耗 Gas 越多
     网络越拥堵, Gas 单价越高(竞价上链)
```

- 💡 **Gas 是 DEX 体验的关键约束**：
  - 以太坊主网拥堵时，一笔 swap 可能花几美元到几十美元 Gas——这就是为什么 DEX 大量迁移到便宜的 L2（见 [3.x](../01-行业全景/1.5-发展简史.md)）。
  - 小额交易在贵的链上不划算（Gas 占比太高）。
- ⚠️ Gas 要用**原生币**付（以太坊用 ETH，Solana 用 SOL）。钱包里没原生币，啥也干不了，连转走其他币都不行。

## 三、出块与确认（为什么 DEX 慢）

```
区块链按"区块"打包交易, 每隔一段时间出一个块:
  以太坊主网: ~12 秒一个块
  Solana:     亚秒级
  L2:         更快

你的交易要"等被打包进区块"才生效 → 这就是 DEX 比 CEX(内存微秒级)慢的根因
```

- **确认数**：交易被打包后，再过几个块才算「最终确定」（防回滚），对照 [CEX 充值确认](../../cex/03-技术架构/3.3-钱包与充提.md)。
- 出块的「离散时间」+ 交易公开排队，正是 MEV（抢跑/夹击）存在的土壤（见 [05](../05-市场与博弈/5.2-MEV详解.md)）。

## 四、EVM 与智能合约

```
EVM(以太坊虚拟机): 运行智能合约的"世界计算机"
  - 智能合约: 部署在链上的代码, 满足条件就自动执行, 不可篡改
  - 用 Solidity 等语言写, 编译成 EVM 字节码
  - "EVM 兼容链"(BSC/Polygon/Arbitrum...)都能跑同样的合约 → 生态可移植
```

- DEX 就是一组智能合约。一旦部署，**代码公开、自动运行、（通常）不可更改**。
- 💡 「不可更改」是双刃剑：没人能偷改规则（好），但**有 bug 也很难修**（坏，见 [06 安全](../06-风险与安全/README.md)）。

### 非 EVM 链
- **Solana**：用 Rust 写合约（叫 program），账户模型不同，高速低费。
- 各链有自己的虚拟机和编程模型，但 DEX 的核心思想相通。

## 五、链上的「公开透明」⭐

```
区块链上的一切都是公开的:
  - 所有交易、所有余额、所有合约代码 → 任何人可查(区块浏览器 Etherscan/Solscan)
  - 你的钱包地址的全部历史一览无余(伪匿名: 不知道是谁, 但能看到地址所有动作)
```

- 好处：DEX 的储备、交易全透明可验证（对照 [CEX 黑盒](../../cex/01-行业全景/1.1-CEX是什么.md)）。
- 代价：**隐私差**（你的持仓和操作全公开），且**交易在打包前就暴露在内存池**——这是 MEV 的前提。

## 把这些串起来：为什么 DEX 是这样 ⭐

```
私钥控制资产 ────────▶ 自托管(不可逆、无人兜底)
Gas + 出块 ─────────▶ 慢、有成本、不适合高频
合约不可篡改 ────────▶ 规则可信但 bug 难修
链上全公开 ─────────▶ 可验证 + 隐私差 + MEV 土壤
EVM 兼容 ──────────▶ 多链生态、可移植、可组合
```

> 理解了底座，后面 AMM 合约、预言机、MEV、安全的种种设计就都有了「为什么」。

## 小结

- 账户=密钥对，私钥/助记词=资产控制权（自托管的根，丢失不可恢复）。
- Gas 是执行燃料、用原生币付、拥堵时贵——是 DEX 体验约束和 L2 迁移的原因。
- 出块的离散时间让 DEX 比 CEX 慢，并构成 MEV 土壤。
- 智能合约公开、自动、（通常）不可改——可信但难修 bug。
- 链上全透明=可验证但隐私差+MEV 前提。
