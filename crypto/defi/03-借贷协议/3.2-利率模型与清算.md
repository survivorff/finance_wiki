# 3.2 利率模型与清算

> 深度：🔴 硬核 ⭐
> 链上借贷的利率怎么算（算法）、清算怎么发生（机器人）。这是 DeFi 借贷的"发动机"和"安全阀"。

## 一、算法利率模型 ⭐

```
传统: 银行/市场定利率
DeFi: 用"算法"根据"资金利用率"自动定利率
利用率(Utilization) = 已借出 / 总存入
  利用率高(钱被借光): 利率高 → 吸引存款 + 抑制借款 → 拉回平衡
  利用率低(钱没人借): 利率低 → 鼓励借款
→ 利率随利用率动态调整, 自动平衡供需
```

### 典型利率曲线（拐点模型）⭐
```
利率
 │                    ╱ 急升(利用率超过"最优点"后陡峭上升)
 │                  ╱
 │        ________╱  ← 拐点(optimal utilization, 如 80%)
 │    ___/  缓升
 │___/
 └──────────────────── 利用率 0% → 100%
设计意图:
  利用率 < 拐点: 利率缓慢上升(正常区间)
  利用率 > 拐点: 利率陡峭飙升 → 强烈激励还款/存款, 防止池子被"借空"
  (池子被借空 → 存款人取不出钱 → 流动性危机)
```

- 💡 **拐点后利率陡升**是关键设计：防止资金池被借光（流动性枯竭），用"陡升的利率"把利用率拉回安全区。这是 DeFi 借贷的流动性保护机制。

## 二、清算机制（Liquidation）⭐ 安全阀

```
清算: 当借款人的抵押率跌破阈值, 强制卖掉抵押品还债
触发: 健康因子(Health Factor) < 1
  健康因子 = (抵押品价值 × 清算阈值) / 借款价值
  → 抵押品跌价 或 借款增值 → 健康因子下降 → < 1 触发清算
```

### 清算流程 ⭐
```
1. 预言机(见 DEX 3.4)报告抵押品价格下跌
2. 借款人健康因子 < 1 → 头寸可被清算
3. "清算人(liquidator)"机器人发现 → 调用清算:
   - 帮忙还掉部分借款
   - 拿走借款人的抵押品 + "清算奖励"(折扣买到抵押品)
4. 协议恢复健康, 清算人获利
→ 清算是"激励驱动"的: 清算人为赚奖励而竞相清算(去中心化, 无需协议自己动手)
```

- 💡 对照 [CEX 强平](../../cex/03-技术架构/3.5-清算与风控.md)、[衍生品 DEX 清算](../../dex/02-产品与机制/2.5-衍生品DEX.md)：
  - CEX：交易所内部强平引擎执行。
  - DeFi：开放的清算人机器人市场，靠奖励激励，竞相执行（去中心化）。
  - **本质都是"抵押不足→强制平仓→防坏账"，区别在谁来执行（中心化引擎 vs 激励驱动的机器人）。**

## 清算的连锁风险 ⚠️

```
极端行情下, 清算可能引发连锁(同 CEX 连环爆仓, 见 CEX 3.5):
  抵押品(ETH)暴跌 → 大量头寸被清算 → 清算抛售 ETH → ETH 更跌
  → 更多清算 → 踩踏(清算瀑布)
+ 链上特有风险:
  - 链拥堵/Gas 飙升 → 清算人来不及清算 → 协议产生坏账
  - 预言机延迟/被操纵 → 错误清算 或 该清算的没清算(见 DEX 3.4)
```

- 💡 **再次出现的踩踏公式**：杠杆 + 拥挤 + 价格冲击 = 踩踏（同 [CEX 爆仓](../../cex/03-技术架构/3.5-清算与风控.md)、[FX 套息平仓](../../../forex/04-汇率原理/4.3-套息交易.md)、[债券 LDI 危机](../../../bonds/06-风险与危机/6.2-债务危机案例.md)）。

## 坏账与协议风险 💡

```
如果清算不及时(抵押品跌太快/链拥堵):
  抵押品价值 < 借款 → 协议产生"坏账"(借款人跑了, 抵押品不够还)
  → 损失由协议/存款人承担
缓冲机制:
  - 储备金/安全模块(协议留一部分作缓冲, 类似 CEX 保险基金, 见 CEX 3.5)
  - 保守的清算阈值(留足缓冲)
```

- 💡 对照 [CEX 保险基金](../../cex/03-技术架构/3.5-清算与风控.md)、[CCP 违约瀑布](../../../stocks/04-清算交收/4.2-中央对手方CCP.md)：DeFi 借贷协议也有"储备金/安全模块"兜底坏账——又是"分层兜底防坏账"的通用设计。

## 利率模型/清算 vs 传统 ⭐

| | DeFi | 传统 |
| --- | --- | --- |
| 利率 | 算法（利用率驱动） | 央行/市场（见 [债券](../../../bonds/05-市场与参与者/5.2-央行与货币政策.md)） |
| 清算执行 | 激励驱动的机器人 | 法律/催收（慢） |
| 清算速度 | 快（链上即时，除非拥堵） | 慢（法律程序） |
| 兜底 | 储备金/安全模块 | 存款保险/资本金 |

## 小结

- 算法利率：利率由"资金利用率"决定（用得多→利率高），拐点后陡升防池子被借空。
- 清算：健康因子<1 触发，由激励驱动的清算人机器人竞相执行（折扣买抵押品赚奖励），去中心化。
- 对照 CEX 强平（内部引擎）：DeFi 用开放的清算人市场，本质同为"抵押不足→强平→防坏账"。
- 连锁风险：清算瀑布（同各市场踩踏）+ 链拥堵/预言机问题导致坏账。
- 协议用储备金/安全模块兜底坏账（同 CEX 保险基金/CCP 违约瀑布）。
