# 2.2 即期与远期

> 深度：🟡 进阶
> FX 最基础的两种交易：即期（现在换）和远期（约定未来换）。

## 即期交易（Spot）

```
即期(Spot): 按"当前汇率"换汇, 通常 T+2 交收(两个工作日后实际钱货两清)
  你看到的"实时汇率"主要就是即期汇率
  例: 现在按 EUR/USD=1.10 把欧元换成美元
```

- 💡 对照 [股票 T+2 交收](../../stocks/04-清算交收/4.1-清算与交收概念.md)：FX 即期也是 T+2（成交后两天才实际交割）。
- 对照 [加密现货即时到账](../../crypto/cex/02-产品体系/2.1-现货交易.md)：FX 即期"成交"是即时的，但"交收"要 T+2（传统金融的延迟交收特征）。

## 远期交易（Forward）⭐

```
远期(Forward): 现在就约定"未来某日按某汇率换汇"
  例: 约定 3 个月后按 EUR/USD=1.105 换汇(不管到时市场价多少)
用途: 锁定未来汇率, 对冲汇率风险
```

### 为什么企业需要远期 ⭐
```
场景: 中国出口商签了合同, 3 个月后收 100 万美元货款
担忧: 3 个月后美元可能贬值(收到的美元换成人民币变少)
解法: 现在就用"远期"锁定 3 个月后的换汇汇率
  → 不管到时美元涨跌, 都按锁定的汇率换 → 消除了汇率不确定性
```

- 💡 这就是 [对冲（hedging）](../06-风险与监管/README.md) 的核心——用远期把"未来的不确定"变成"现在的确定"。
- 对照 [加密交割合约](../../crypto/cex/02-产品体系/2.2-合约衍生品.md)：都是"锁定未来价格"，逻辑一致。

## 远期汇率怎么定：利率平价 ⭐⭐

远期汇率不是"猜"出来的，而是由**两种货币的利率差**决定：

```
远期汇率 ≈ 即期汇率 × (1 + 报价货币利率) / (1 + 基础货币利率)

直觉: 如果美元利率高于欧元利率
  → 持有美元能赚更多利息
  → 为了"无套利", 远期的美元必须比即期"贵一点"来抵消利息优势
  → 否则大家都借欧元换美元吃利差(无风险套利)
```

> ⭐ **这是 FX 最重要的原理之一：远期汇率由利率差锁定（覆盖利率平价 CIP）。**
> 详细推导见 [04 汇率原理](../04-汇率原理/4.2-利率平价与购买力平价.md) 和 [08 深水区](../08-深水区/8.1-远期掉期定价.md)。
> 💡 这和 [CEX 资金费率/期现基差](../../crypto/cex/08-深水区/8.2-永续合约数学全解.md) 是同一类"无套利定价"——市场不允许免费午餐，远期价格被套利锁死。

## 远期点（Forward Points）

```
远期点 = 远期汇率 − 即期汇率
  正值(升水): 远期比即期贵
  负值(贴水): 远期比即期便宜
由利率差决定(高息货币远期贴水, 低息货币远期升水)
```

## NDF（无本金交割远期）💡

```
NDF(Non-Deliverable Forward): 用于有资本管制、不能自由兑换的货币
  到期不实际交割本金, 只结算"约定汇率"和"实际汇率"的差额(用美元)
  例: 离岸交易人民币、印度卢比等的远期
→ 绕开资本管制对冲汇率风险
```

- 💡 对照 [加密永续合约的现金结算](../../crypto/cex/02-产品体系/2.2-合约衍生品.md)：NDF 也是"只结差额不交本金"，思路一致。

## 小结

- 即期（Spot）：按当前汇率换汇，T+2 交收（同股票的延迟交收）。
- 远期（Forward）：现在约定未来换汇汇率，用于对冲汇率风险（企业核心需求）。
- 远期汇率由两种货币的利率差决定（覆盖利率平价）——是无套利定价，同加密期现基差。
- NDF（无本金交割远期）用于受管制货币，只结差额不交本金（同加密现金结算合约）。
