# 3.5 清算与风控系统

> 深度：🔴 硬核
> 撮合只负责「匹配」，匹配之后的「改余额、管风险、防穿仓」都是清算与风控的事。
> 这是合约业务的安全阀，没有它，一次极端行情就能让交易所产生巨额坏账。

## 一、清算 / 结算（Clearing & Settlement）

### 撮合 ≠ 结算
- **撮合引擎**：只负责「谁和谁、以什么价、成交多少」，输出成交记录。它不碰余额。
- **清算服务**：拿着成交记录，去**实际改双方的账户余额、扣手续费**。

```
撮合引擎 ──成交记录──▶ 清算服务 ──▶ 账户系统(改余额、扣费、复式记账)
  (匹配)                 (结算)         (记账)
```

这种「**撮合与清算分离**」的设计，让撮合引擎保持极简极快，把复杂的账务逻辑交给清算服务。

- **现货结算**：成交后立即结算双方资产（即时交收）。
- **合约结算**：维护保证金、实时计算盈亏（PnL）、周期结算资金费率。

## 二、合约风控的关键价格（先理解这个）

合约风控不能用「最新成交价」，否则容易被操纵。所以有两个特殊价格：

### 指数价（Index Price）
- 取**多家现货交易所**该币种价格的加权平均。
- 目的：代表「市场公允价」，单一交易所被操纵也影响不大。

### 标记价（Mark Price）⭐
- 基于指数价 + 合理基差（资金费率相关）推算出来的「公允标记价」。
- **用于计算未实现盈亏、触发强平。**

> ⭐ **为什么不用最新成交价触发强平**：
> 如果用最新成交价，恶意玩家可以在深度薄的时刻用一笔大单把价格瞬间砸到极低（**插针**），
> 触发大量无辜用户爆仓，自己低位接盘获利。用「多所加权的标记价」就能极大削弱这种操纵。

## 三、强平引擎（Liquidation Engine）⭐ 合约的安全阀

### 触发逻辑
```
实时监控每个合约仓位的保证金率:
保证金率 = 账户权益 / 持仓所需保证金

当 保证金率 < 维持保证金率 → 触发强平
```

### 强平流程
```
1. 标记价驱动，实时扫描所有仓位的保证金率
2. 某仓位触及维持保证金 → 系统接管该仓位
3. 强制平仓(在市场上把仓位平掉)
4. 平仓结果分情况处理 ↓
```

## 四、防穿仓三道防线 ⭐⭐

**穿仓**：价格剧烈波动，用户亏损超过其保证金（账户变负）。这个窟窿谁来补？

```
第1道：强平引擎
  └─ 在用户亏光保证金"之前"就接管平仓，争取平在还有剩余保证金时
     │
第2道：保险基金(Insurance Fund)
  ├─ 强平有"盈余"(平在维持保证金之上) → 盈余存入保险基金
  └─ 强平"穿仓"(平价低于破产价，亏损超保证金) → 从保险基金拿钱填窟窿
     │
第3道：自动减仓 ADL(Auto-Deleveraging)
  └─ 极端行情下保险基金也被掏空 → 系统强制平掉"对手方盈利最多、杠杆最高"的仓位
     用他们的盈利来填补穿仓亏损
```

### 通俗解释三道防线
- **强平**：在你快亏光时强制帮你（和系统）止损，别拖到欠债。
- **保险基金**：一个公共资金池，平时靠强平盈余积累，关键时刻兜底穿仓。
- **ADL**：最后手段——保险基金都不够时，强行让「赚最多的对手方」提前下车，用他们的利润补窟窿。ADL 是用户体验最差的机制（赚钱也被强制平仓），所以是最后防线。

> ⭐ **设计目标**：保证**交易所不替用户的亏损买单、系统不产生坏账（socialized loss 最小化）**。
> 这套机制让交易所在剧烈波动中也能维持「账平」。

### 连环爆仓（Liquidation Cascade）
```
价格下跌 → 多头爆仓被强平(市价卖出) → 卖压加大 → 价格更跌 → 更多多头爆仓 → ...
```
高杠杆 + 深度不足时，强平会自我强化形成踩踏。这是加密市场「插针」「瀑布」的技术根源。

## 五、实时风控（Risk Control）

除了合约强平，交易所还有一整套实时风控：

| 风控类型 | 防什么 |
| --- | --- |
| **交易风控** | 自成交、操纵、异常下单 |
| **账户风控** | 盗号、异地登录、改密后提币 |
| **反洗钱(AML)** | 可疑资金、制裁地址、链上追踪 |
| **提币风控** | 黑名单地址、大额审批、冷静期 |
| **API 风控** | 频率限制、异常调用 |
| **市场风控** | 持仓限额、杠杆限制、风险准备金监控 |

### 链上风控（加密特有）
- 接入链上分析工具（如 Chainalysis、TRM），筛查充值资金是否来自黑客、混币器、制裁地址。
- 标记/冻结涉案资金（这也引发「中心化交易所能冻结资产」的中心化争议）。

## 六、风控系统的技术特点

- **实时性要求极高**：强平判断要在毫秒级完成，价格波动剧烈时仓位扫描压力巨大。
- **和撮合/清算紧耦合**：风控触发强平 → 走撮合平仓 → 清算结算，是一条实时闭环。
- **规则引擎 + 流式计算**：用规则引擎配置风控策略，用流式计算（如 Flink）实时处理交易流。

## 小结

- 撮合与清算分离：撮合只匹配，清算改余额、管保证金、结算盈亏。
- 合约用**指数价/标记价**（多所加权）防插针操纵。
- 强平引擎是合约安全阀，**三道防线**（强平→保险基金→ADL）防穿仓、防坏账。
- 连环爆仓是「瀑布行情」的技术根源。
- 实时风控覆盖交易、账户、AML、提币、API，并接入链上分析。
