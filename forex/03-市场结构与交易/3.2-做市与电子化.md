# 3.2 做市与电子化

> 深度：🔴💡 内幕
> FX 做市的机制，以及电子化带来的新现象（算法、"最后观望"等争议）。

## FX 做市（同源但有特色）

FX 做市的核心数学（库存风险、逆向选择、价差补偿）和 [股票 8.3](../../stocks/08-深水区/8.3-做市与执行算法.md)、[CEX 8.7](../../crypto/cex/08-深水区/8.7-做市与市场微观结构.md) 完全相同。这里讲 FX 特色。

### FX 做市的特点
```
- 没有中央交易所 → 做市银行直接对客户/对彼此报价
- 大银行用算法自动报价(几乎全电子化)
- 做市银行要管理"跨多个货币对"的库存风险
- 流动性高度集中在几家顶级银行
```

## 电子化的演进 💡

```
过去: 交易员电话喊价、聊天软件议价(人工)
现在: 几乎全电子化
  - 单一银行平台: 银行给客户自己的报价系统
  - 多银行平台/ECN: 聚合多家银行报价(EBS、Reuters、各种 ECN)
  - 算法做市: 银行用算法自动报价、对冲
  - 算法执行: 客户用 TWAP/VWAP 等算法执行大单(同 股票 8.3)
```

## "最后观望"（Last Look）⚠️ FX 独有的争议机制

```
Last Look: FX 做市商收到客户订单后, 有"最后一刻拒绝成交"的权利(几毫秒内)
  → 做市商可以在"价格对自己不利"时拒绝(重新报价)
  → 名义上是防"被套利/防延迟", 实际可能损害客户
争议: 做市商用它"挑单"(对自己有利就成交, 不利就拒绝)
→ 是 FX 特有的、被诟病的灰色机制(监管在关注)
```

- 💡 **这是 FX 独有的现象**，本质是「做市商利用信息/时间优势挑选订单」。
- 💡 对照 [DEX MEV](../../crypto/dex/05-市场与博弈/5.2-MEV详解.md) 和 [股票抢跑](../../stocks/06-监管与风险/6.2-市场操纵与违规.md)：都是"用信息/时间优势占客户便宜"的变体。**只要市场有信息和时间不对称，这类问题就以不同形态出现。**

## FX 的高频交易

```
FX 也有高频交易(同 股票 7.2):
  - 延迟套利(跨平台价差)、电子做市、统计套利
  - 同样的速度竞赛(托管、低延迟)
→ FX 高频和股票/加密的高频是同一套技术军备竞赛
```

## "聚合"与碎片化的张力 💡

```
FX 流动性分散在: 多家银行平台 + 多个 ECN + 期货交易所
→ 和 股票碎片化(见 股票 3.5) 类似
→ 需要"聚合器"(aggregator)汇总各家报价, 找最优执行
→ 大机构用聚合器 + 智能订单路由(同 股票 SOR、DEX 聚合器)
```

- 💡 **又见碎片化 + 聚合**：FX、股票、DEX 都因"多场所"而碎片化，都靠"聚合 + 路由 + 套利"解决。这是跨市场的共同结构性主题。

## FX 操纵丑闻 ⚠️

```
历史: 多家大银行因"操纵 FX 基准汇率"被罚巨款
  手法: 交易员在聊天室串通, 在"定盘(fixing)"时点集中下单影响基准汇率
  → 类似 股票/加密的"收盘价操纵"(见 股票 6.2、加密 5.4)
→ 引发对 FX 基准(如 WM/Reuters fixing)和做市行为的监管收紧
```

## 小结

- FX 做市数学和股票/加密相同，特点是无中央交易所、全电子化、大银行集中、跨货币对库存管理。
- "最后观望"（Last Look）是 FX 独有争议机制——做市商可最后一刻拒单，本质是用时间优势挑单（类似 MEV/抢跑）。
- FX 同样有高频交易和碎片化+聚合（同股票/DEX）。
- FX 曾爆操纵基准汇率的丑闻（定盘时点串通下单），类似收盘价操纵，引发监管收紧。
