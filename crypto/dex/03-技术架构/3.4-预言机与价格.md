# 3.4 预言机与价格

> 深度：🔴 硬核 ⭐
> 区块链是个「封闭世界」，它不知道外面的价格。预言机就是把外部数据喂进链上的桥梁——也是 DeFi 最经典的攻击面。

## 核心问题：链不知道「真实价格」

```
智能合约要清算、要给衍生品定价 → 需要知道"ETH 现在多少钱"
但区块链是封闭的, 合约无法主动访问外部世界(交易所价格、API...)
→ 需要"预言机(Oracle)"把外部数据搬上链
```

> 💡 这叫「**预言机问题（Oracle Problem）**」：去中心化的链如何可信地获取链外数据？这是 DeFi 的根本性挑战之一。

## 谁需要预言机

| 场景 | 为什么 |
| --- | --- |
| 借贷协议清算 | 要知道抵押物价值是否跌破清算线 |
| 衍生品 DEX（GMX 型） | 直接用预言机价格成交（见 [2.5](../02-产品与机制/2.5-衍生品DEX.md)） |
| 稳定币 | 维持锚定需要价格参考 |
| 合成资产 | 跟踪标的价格 |

> ⚠️ **注意**：纯 AMM 现货 DEX（如 Uniswap swap）**本身不需要预言机**——它的价格是池子比例内生的。预言机主要是**借贷、衍生品、以及"把 DEX 价格当预言机用"**的场景需要。

## 危险：直接用 DEX 现货价做预言机 ⚠️⭐

一个经典的致命错误：

```
某协议直接读取"Uniswap 池子当前价格"作为预言机
   ↓
攻击者用【闪电贷】(见 1.4)借巨款:
   1. 闪电贷借入巨量资金
   2. 在那个池子里巨额买入 → 瞬间把池子价格拉到天上
   3. 用被操纵的虚高价格, 去那个协议借出/套利
   4. 归还闪电贷, 卷走利润
   全在一笔交易内完成 → 协议被掏空
```

- 💡 **这是 DeFi 史上最经典的攻击之一**：「闪电贷 + 价格操纵」组合拳。
- 根因：**单一池子的瞬时价格极易被大额交易操纵**，绝不能直接当预言机。

## 解法一：TWAP（时间加权平均价格）⭐

```
不用"瞬时价格", 用"过去一段时间的加权平均价":
  TWAP = 过去 N 个区块/时间窗口的价格平均
  → 攻击者要操纵 TWAP, 必须在【整个时间窗口】维持操纵价
  → 成本极高(要持续投入巨资且承受套利者反向收割) → 操纵不划算
```
- Uniswap V2/V3 内置了 TWAP 预言机功能。
- **代价**：TWAP 是滞后的（用历史均价），剧烈行情时不够实时。

## 解法二：去中心化预言机网络（Chainlink）⭐

```
Chainlink 等: 一组独立节点各自从多个数据源(多家 CEX/DEX)取价
            → 聚合(去极值、取中位数) → 上链
            → 多节点 + 多数据源 → 单点操纵/故障不影响结果
```
- **优点**：抗操纵（要同时骗多个独立节点和多个数据源极难）、覆盖广。
- **代价**：有更新延迟和成本；本身是一层需要信任的基础设施（节点是否真去中心化、数据源是否可靠）。
- 是目前最主流的预言机方案。

## 解法三：低延迟/拉取式预言机（Pyth 等）

```
Pyth 等: 由专业机构(交易所、做市商)直接提供高频价格
        用"拉取(pull)"模式: 需要时把签名价格数据拉上链
        → 更低延迟、更适合衍生品
```
- 适合需要高频价格的衍生品 DEX。
- 权衡：数据来自专业机构（更快但更依赖这些数据提供方）。

## 预言机的风险全景 ⚠️

| 风险 | 说明 |
| --- | --- |
| **价格操纵** | 用闪电贷等操纵被读取的价格源 |
| **预言机延迟** | 行情剧变时喂价滞后 → 清算不及时/错误成交 |
| **预言机宕机/失灵** | 数据源挂了 → 依赖它的协议瘫痪或乱套 |
| **中心化隐忧** | 预言机网络本身的去中心化程度 |
| **喂价错误** | 数据源 bug 喂出离谱价格 → 连锁清算 |

> 💡 **很多 DeFi 大额被盗事件，根因都是预言机被操纵或失灵。**
> 预言机是「链上世界和链外现实之间的脆弱接缝」——它把现实的价格搬进来，也把现实的风险（和攻击面）带了进来。

## 对照 CEX 的标记价 ⭐

还记得 [CEX 的标记价/指数价](../../cex/03-技术架构/3.5-清算与风控.md) 吗？也是「多所加权防操纵」——
**DEX 的预言机和 CEX 的指数价，解决的是同一个问题：如何获得一个不易被操纵的公允价格用于清算。**
只是 CEX 在内部用多所数据加权，DEX 用去中心化预言机网络。又一个「同一问题，不同范式」的例子。

## 小结

- 区块链封闭，预言机是把链外数据（尤其价格）喂上链的桥梁，是「预言机问题」。
- 借贷/衍生品需要预言机；纯 AMM swap 不需要（价格内生）。
- 致命错误：直接用单池瞬时价做预言机 → 被「闪电贷+操纵」攻击。
- 解法：TWAP（时间加权抗操纵但滞后）、Chainlink（多节点多源聚合）、Pyth（低延迟拉取式）。
- 预言机是链上链下的脆弱接缝，是 DeFi 经典攻击面；与 CEX 标记价解决同一问题。
