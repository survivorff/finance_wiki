# 3.3 交收与 CLS

> 深度：🔴 硬核 ⭐
> FX 交收有一个独特的、致命的风险——「赫斯塔特风险」，以及为解决它而生的 CLS 银行。这是 FX 版的 DVP。

## 问题：FX 交收的「赫斯塔特风险」⭐

```
FX 交易要交换两种货币, 但分属不同国家、不同时区、不同支付系统:
  你付欧元(欧洲时段) ←→ 对方付美元(美国时段)
  → 两笔支付不在同一时刻发生!
风险: 你已经付出了欧元, 但对方在付美元之前破产了
  → 你欧元付了, 美元没收到 → 损失全部本金!
```

### 赫斯塔特风险（Herstatt Risk）的由来 💡
```
1974 年: 德国 Herstatt 银行被关闭
  当时: 一些对手方已经付了德国马克给 Herstatt
        但 Herstatt 关闭时还没付出对应的美元(纽约时段还没到)
  → 这些对手方损失了已付出的马克 → 著名的"赫斯塔特风险"
→ FX 交收的"时间差风险"从此有了名字, 并催生了解决方案
```

- 💡 这本质是 [股票 DVP（券款对付）](../../stocks/04-清算交收/4.1-清算与交收概念.md) 要解决的同一个问题——**"给了钱没收到货"的交收风险**，只是 FX 因跨时区更严重。

## 解决方案：CLS 银行（PvP）⭐

```
CLS(Continuous Linked Settlement): 专门做 FX 交收的全球性机构
核心机制 — PvP(Payment versus Payment, 支付对支付):
  两笔货币支付"同时发生, 要么都成功要么都不发生"
  → 你付欧元 和 对方付美元 在 CLS 内同步结算
  → 消除"付了一边没收到另一边"的赫斯塔特风险
```

- 💡 **PvP 之于 FX = [DVP 之于股票](../../stocks/04-清算交收/4.1-清算与交收概念.md) = [原子结算之于 DEX](../../crypto/dex/01-行业全景/1.4-DeFi生态位.md)**：
  - 股票：券款对付（DVP），股票和钱同时换。
  - FX：支付对支付（PvP），两种货币同时换。
  - DEX：链上原子交换，要么全成要么回滚。
  - **三者解决的是同一个根本问题：如何让"双向交换"原子化，消除"给了一边没收到另一边"的风险。** 这是金融交收的黄金原则，跨所有市场。

## CLS 的轧差 💡

```
CLS 也做多边轧差(同 股票 CCP, 见 股票 8.2):
  把会员当天的多笔 FX 交易轧差成净额结算
  → 大幅减少实际需要流动的资金量
```

## FX 交收的现状

```
- CLS 覆盖主要货币的大部分银行间 FX 交收
- 但仍有部分 FX 交收没走 CLS(小币种、某些对手) → 仍有赫斯塔特风险残留
- 零售 FX: 散户不实际交收货币(多是差价合约 CFD, 见 06), 不涉及这个问题
```

## 对照其他市场的交收担保 ⭐

| 市场 | 交收风险解法 | 机制名 |
| --- | --- | --- |
| 股票 | CCP + DVP | 中央对手方 + 券款对付 |
| FX | CLS + PvP | 支付对支付 |
| DEX | 区块链原子性 | 链上原子交换 |
| CEX | 内部记账（无外部交收） | 数据库即时改余额 |

> 💡 **金融基础设施的演化，很大程度就是"如何安全地完成双向交换"的演化**：
> 从赫斯塔特银行倒闭的惨痛教训（FX），到股票的 DVP，到 DEX 用区块链原子性"一劳永逸"地解决——
> **DEX 的原子结算，某种意义上是这个几十年问题的"终极优雅解"**（用代码保证原子性，不需要 CLS/CCP 这样的中介机构）。

## 小结

- FX 交收因跨时区，有"赫斯塔特风险"——付了一种货币、对方破产没付另一种，损失本金（1974 Herstatt 银行倒闭的教训）。
- CLS 银行用 PvP（支付对支付）让两种货币同步结算，消除该风险。
- PvP（FX）= DVP（股票）= 原子结算（DEX），都解决"双向交换原子化"的根本问题。
- DEX 的链上原子结算是这个几十年问题的终极优雅解（用代码替代 CLS/CCP 中介）。
