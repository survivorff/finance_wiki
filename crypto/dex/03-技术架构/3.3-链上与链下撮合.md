# 3.3 链上撮合 vs 链下撮合

> 深度：🔴 硬核
> 「撮合放哪」是 DEX 架构最核心的取舍。这篇把几种方案讲透。
> 前置：[CEX 8.5 架构对比](../../cex/08-深水区/8.5-真实架构案例拆解.md)。

## 核心矛盾

```
区块链: 安全、透明、去中心化, 但 慢(出块) + 贵(Gas) + 公开(MEV)
交易撮合: 需要 快 + 频繁(挂撤单) + 私密(不暴露订单)

→ 把"撮合"全放链上, 和区块链的特性天然冲突
→ 不同 DEX 用不同方式调和这个矛盾
```

## 四种方案光谱

```
全链上 ◀──────────────────────────────────────▶ 偏链下
 AMM           链上订单簿        链下撮合+链上结算      专用链
(Uniswap)     (早期, 已少)     (0x/早期dYdX)        (dYdX v4/Hyperliquid)
```

## 方案一：AMM（绕开撮合问题）⭐

- **思路**：干脆不要订单簿和撮合，用公式定价（见 [2.1](../02-产品与机制/2.1-AMM是什么.md)）。
- **优点**：完全链上、合约极简、无需对手方、Gas 可控。
- **缺点**：滑点、无常损失、资本效率（用集中流动性缓解）。
- 这是「绕过」撮合难题的最成功方案，统治了现货 DEX。

## 方案二：纯链上订单簿（理论纯粹，实践艰难）

- **思路**：把订单簿和撮合完全放链上。
- ⚠️ **问题**：每次挂单、撤单、撮合都是链上交易，要 Gas、等出块——慢、贵、做市商根本没法高频报价。EtherDelta 的失败（见 [1.5](../01-行业全景/1.5-发展简史.md)）。
- 只在极高性能链上才勉强可行。

## 方案三：链下撮合 + 链上结算（混合）

```
撮合(慢、频繁的部分) 放链下: 中继器(relayer)维护订单簿, 做市商免 Gas 挂撤单
结算(资产交割) 放链上:      撮合成功后, 把成交提交链上结算(自托管, 可验证)
```
- **代表**：0x 协议、早期 dYdX。
- **优点**：撮合快（链下）+ 结算可信（链上）+ 自托管。
- **缺点**：中继器是个半中心化环节（撮合不完全去中心化）；做市商签名订单的机制较复杂。

## 方案四：专用高性能链（当前前沿）⭐

```
干脆为交易"定制一条链", 让链本身够快, 就能把订单簿放链上:
  dYdX v4:     自建 Cosmos 应用链, 验证者各维护链下内存订单簿, 出块时链上结算
  Hyperliquid: 自建 L1 (HyperBFT 共识), 链上订单簿, 体验逼近 CEX
```
- **思路**：不在通用链上挣扎，而是把「交易所」做成「一条专门的链」。
- **优点**：性能接近 CEX + 保持去中心化结算/可验证。
- **缺点**：自建链的去中心化程度、验证者数量、抗审查性需要审视；生态相对独立。
- 💡 这是 [CEX 撮合引擎](../../cex/03-技术架构/3.2-撮合引擎.md) 思想和区块链的融合，详见 [CEX 8.5](../../cex/08-深水区/8.5-真实架构案例拆解.md)。

## 四方案对比表 ⭐

| 方案 | 撮合位置 | 速度 | 去中心化 | 代表 | 适合 |
| --- | --- | --- | --- | --- | --- |
| AMM | 无撮合(公式) | 中(受出块) | 高 | Uniswap | 现货、长尾 |
| 纯链上订单簿 | 全链上 | 慢 | 高 | (已少) | 几乎不用 |
| 链下撮合+链上结算 | 链下中继器 | 快 | 中 | 0x | 限价单、专业 |
| 专用链 | 链上(专用链) | 快 | 中高 | Hyperliquid/dYdX v4 | 衍生品、高频 |

## 没有标准答案 ⭐

> 「撮合放哪」本质是在 [CEX 8.5 的不可能三角](../../cex/08-深水区/8.5-真实架构案例拆解.md)（性能 / 去中心化 / 功能）里选位置：
> - 要极致去中心化和简单 → AMM（牺牲撮合体验）。
> - 要订单簿体验 → 专用链或链下撮合（牺牲一点去中心化纯度）。
>
> 现货领域 AMM 赢了；衍生品领域专用链（Hyperliquid）正在赢。不同业务，最优解不同。

## 小结

- 核心矛盾：撮合要快/频繁/私密，区块链慢/贵/公开，需要调和。
- 四种方案：AMM（绕开撮合，现货主流）、纯链上订单簿（太慢，淘汰）、链下撮合+链上结算（混合，半中心化）、专用高性能链（前沿，衍生品主流）。
- 本质是在「性能/去中心化/功能」不可能三角里选位置，不同业务最优解不同。
