# 8.2 永续合约数学全解

> 深度：🔴🔴 极硬核
> 把永续合约的每一个数字——保证金、强平价、资金费率、PnL、ADL——都用公式讲清楚。
> 前置：[2.2 合约衍生品](../02-产品体系/2.2-合约衍生品.md)、[3.5 清算与风控](../03-技术架构/3.5-清算与风控.md)。

> ⚠️ 各交易所公式细节有差异（尤其维持保证金分档、资金费率构成）。本文给出**通用形式**和**典型实现**，帮你理解机制，实操以具体交易所文档为准。

## 符号约定

| 符号 | 含义 |
| --- | --- |
| `Q` | 仓位数量（合约张数 → 折算成币的数量） |
| `P_entry` | 开仓均价 |
| `P_mark` | 标记价（用于盈亏和强平，见 [3.5](../03-技术架构/3.5-清算与风控.md)） |
| `L` | 杠杆倍数 |
| `IM` | 起始保证金率 (Initial Margin rate) = 1/L |
| `MM` | 维持保证金率 (Maintenance Margin rate)，通常 < IM |
| `Margin` | 实际投入的保证金 |

## 一、PnL（盈亏）

### 未实现盈亏（Unrealized PnL）
对**线性合约（U 本位）**，做多：
```
uPnL = Q × (P_mark − P_entry)
```
做空：
```
uPnL = Q × (P_entry − P_mark)
```

### 反向合约（币本位）的区别 ⭐
币本位合约用「币」结算，PnL 是非线性的（因为合约面值以 USD 计，但用币交割）：
```
做多 uPnL(以币计) = Q_usd × (1/P_entry − 1/P_mark)
```
> 💡 这就是为什么币本位做多 BTC、BTC 还涨，你赚的「币」却比线性合约少——非线性的「凸性」在和你作对/帮你。这是币本位最反直觉的点。

## 二、保证金

### 起始保证金（开仓需要多少钱）
```
Initial Margin = (Q × P_entry) / L = 名义价值 / 杠杆
```
例：开 10000 USDT 名义、10x 杠杆 → 需要 1000 USDT 起始保证金。

### 维持保证金（不被强平的最低线）
```
Maintenance Margin = Q × P_mark × MM
```
- `MM`（维持保证金率）通常远小于 `IM`。比如 10x（IM=10%）的仓位，MM 可能是 0.5%。
- ⭐ **MM 通常是分档累进的（tiered）**：仓位越大，MM 率越高（大仓位风险大，要求更多保证金）。这就是「仓位分级（risk tier）」机制。

### 保证金率（账户健康度）
```
Margin Ratio = 维持保证金 / 账户权益
账户权益 (Equity) = 钱包余额 + 未实现盈亏(uPnL)

当 Margin Ratio ≥ 100%（即权益 ≤ 维持保证金）→ 触发强平
```

## 三、强平价格（Liquidation Price）⭐ 最关心的数字

### 逐仓做多的强平价（简化通用形式）
强平发生在「权益恰好等于维持保证金」时。逐仓下权益 = 保证金 + uPnL：

```
Margin + Q×(P_liq − P_entry) = Q × P_liq × MM
```
解出 `P_liq`：
```
            P_entry − Margin/Q
P_liq = ─────────────────────────
              1 − MM
```

代入 `Margin = Q×P_entry/L`（逐仓起始保证金）：
```
              P_entry × (1 − 1/L)
P_liq(多) = ──────────────────────
                  1 − MM
```

### 直观结论 ⭐
```
忽略 MM 时近似:  P_liq(多) ≈ P_entry × (1 − 1/L)
```
- 10x 杠杆做多 → 价格跌约 **10%** 就接近强平。
- 100x 杠杆做多 → 价格跌约 **1%** 就强平。
- 做空对称：`P_liq(空) ≈ P_entry × (1 + 1/L)`。

> 💡 **这就是高杠杆的致命之处的数学本质**：杠杆 L 越大，`1/L` 越小，强平价离开仓价越近，
> 一点点波动就触及。100x 不是「100 倍收益」，是「1% 反向波动就归零」。

### 全仓（Cross）的复杂性
全仓下，账户**所有余额和其他仓位的盈亏**都算入权益，强平价会随其他仓位变化而浮动，计算复杂得多（要把整个账户作为一个整体算）。逐仓是隔离的，好算；全仓抗波动强但牵一发动全身。

## 四、资金费率（Funding Rate）⭐

### 作用与结算
永续合约靠资金费率把合约价锚定现货价（见 [2.2](../02-产品体系/2.2-合约衍生品.md)）。

```
资金费用 = 仓位名义价值 × Funding Rate
        = (Q × P_mark) × F

F > 0:  多头付给空头   （多头情绪过热时）
F < 0:  空头付给多头
每 8 小时结算一次（典型），交易所不收取这部分（多空之间转移）
```

### 资金费率怎么算（典型构成）
```
F = Premium Index(溢价指数) + clamp(Interest Rate − Premium Index, ±clamp)
```
拆开看：
- **Premium Index（溢价）**：永续合约价相对现货指数价的偏离程度。
  ```
  Premium = (合约买卖中价 − 指数价) / 指数价   (加权平滑)
  ```
- **Interest Rate（利率）**：两种计价货币的利差，通常是个小固定值（如 0.01%/8h）。
- **clamp**：把利率与溢价的差限制在一个范围内，避免极端值。

> 💡 **套利者视角**：当 F 长期为正（牛市多头狂热），「现货买入 + 永续做空」delta 中性组合可持续收取资金费。
> 这正是 [5.3 资金费率套利](../05-市场与博弈/5.3-典型玩法与套利.md) 的数学来源——你赚的是「多头愿意为杠杆多头付的溢价」。

## 五、强平流程的数学（接管之后）

```
触发强平(权益 ≤ 维持保证金)
   │
   ▼
以"破产价 P_bankruptcy"为界平仓:
   破产价 = 权益归零时的价格 = P_entry × (1 − 1/L)   (做多, 忽略费用)
   │
   ├── 平仓价 优于 破产价 → 剩余保证金 → 注入保险基金
   └── 平仓价 劣于 破产价 → 穿仓亏损 → 保险基金 兜底
                              └── 保险基金不足 → ADL 自动减仓
```

### ADL（自动减仓）的排序数学
保险基金被掏空时，系统要选「谁被强制平仓来填窟窿」。按 **ADL 排名** 选择对手方：
```
ADL 优先级 ∝ 盈利率 × 有效杠杆
（盈利越多、杠杆越高的对手方，越先被减仓）
```
- 交易所 App 里那个「ADL 灯」就是显示你在队列里的危险程度。
- 💡 ADL 是最后防线，代价是「赚钱的人被强制平仓」，体验极差，所以排在保险基金之后。

## 六、一个完整算例 ⭐

```
开仓: 做多 1 BTC, P_entry = 60000, 杠杆 10x, 逐仓
   起始保证金 = 60000 / 10 = 6000 USDT
   假设 MM = 0.5%

强平价 P_liq(多) = 60000 × (1 − 0.1) / (1 − 0.005)
               = 54000 / 0.995
               ≈ 54271 USDT
   → 价格从 60000 跌到约 54271 (约 −9.5%) 就强平，符合"10x 约跌10%爆"的直觉。

若标记价现在 = 61000:
   uPnL = 1 × (61000 − 60000) = +1000 USDT
   账户权益 = 6000 + 1000 = 7000 USDT

资金费率 F = +0.01% (此刻):
   作为多头你要付 = 1 × 61000 × 0.0001 = 6.1 USDT 给空头
```

## 七、为什么这些公式对交易所是「安全阀」

> 把第五节连起来：**强平价 < 破产价** 这个缓冲区间（由 MM 提供）是关键。
> 只要强平引擎能在「权益还没归零（到破产价）之前」就平掉仓位，
> 平仓的剩余保证金就进保险基金，系统不产生坏账。
> MM 越大、强平越早，缓冲越厚，但用户「可用杠杆空间」越小——这是风控与体验的权衡。

## 小结

- PnL：线性合约 `Q×(P_mark−P_entry)`；币本位是非线性的（凸性）。
- 起始保证金 = 名义/杠杆；维持保证金 = 名义×MM（分档累进）。
- 强平价近似 `P_entry×(1∓1/L)`——这是高杠杆致命性的数学本质。
- 资金费率 = 溢价 + 利率项，每 8h 多空互付，是套利的来源。
- 强平用破产价为界，缓冲区间→保险基金→ADL（按盈利×杠杆排序）是三道防线的数学实现。
