# 1.2 CEX vs DEX

> 深度：🟢 小白友好
> 先建立对照，第二阶段（DEX 模块）会深入。这里目的是让你理解 CEX 的「定位」。

## 一句话区别

- **CEX**：把钱交给一家公司，享受快和方便，承担「公司跑路/作恶」的风险。
- **DEX**：钱始终在自己钱包里，自己掌控，但要承担「操作复杂、合约漏洞、速度慢」的代价。

本质是 **「便利 vs 自主」** 的取舍。

## 全面对照表

| 维度 | CEX 中心化交易所 | DEX 去中心化交易所 |
| --- | --- | --- |
| **资产保管** | 交易所托管（Not your keys） | 用户自持私钥 |
| **撮合方式** | 中心化撮合引擎 / 订单簿 | 链上智能合约 / AMM 自动做市 |
| **速度** | 极快（内存撮合，微秒级） | 受限于区块链出块（秒级以上） |
| **成本** | 手续费（0.1% 量级） | 手续费 + 链上 gas 费 |
| **法币通道** | 有（可直接出入金） | 一般没有（需第三方） |
| **KYC 实名** | 通常强制 | 一般无需，匿名 |
| **门槛** | 低，注册即用 | 高，要会用钱包、懂 gas |
| **资产种类** | 主流币 + 精选币 | 任何人都能上的长尾币（鱼龙混杂） |
| **典型风险** | 平台跑路、被黑、挪用资金 | 合约漏洞、被夹(MEV)、无常损失、貔貅盘 |
| **透明度** | 黑盒（撮合不可见） | 全透明（链上可查） |
| **代表** | Binance, Coinbase, OKX | Uniswap, Curve, dYdX, Hyperliquid |

## 两种撮合范式（核心差异）

### CEX：订单簿撮合
买卖双方挂单，引擎匹配价格。需要做市商提供流动性。（详见 [03 技术架构](../03-技术架构/README.md)）

### DEX：AMM 自动做市商
没有订单簿，用一个数学公式（如 `x*y=k`）和一个「资金池」来定价。
任何人往池子里存两种币就能当「做市商」赚手续费。（DEX 模块会深入）

```
CEX 订单簿:  买家挂单 ←撮合→ 卖家挂单     （需要对手方）
DEX AMM:     用户 ←→ 资金池(按公式自动定价)  （和池子交易，不需要对手方）
```

## 为什么两者会长期共存

- **CEX 的不可替代**：法币出入金、深流动性、高频交易、衍生品体验、客服兜底。
- **DEX 的不可替代**：自托管安全、抗审查、长尾资产、链上原生（DeFi 可组合性）。

> 💡 **行业现状**：交易量上 CEX 仍远大于 DEX，但 DEX 份额持续上升，尤其在衍生品领域（如 Hyperliquid 崛起）。
> 头部 CEX 也在反向布局 DEX/Web3 钱包（如 Binance Web3 Wallet、Coinbase 的 Base 链），界限在模糊。这个趋势在 [07 未来发展](../07-未来发展/README.md) 详谈。

## ⚠️ 常见误区

- **「DEX 一定比 CEX 安全」**：不一定。DEX 没有跑路风险，但合约漏洞、貔貅盘（只能买不能卖的骗局）、私钥丢失同样能让你血本无归，而且没有客服能帮你。
- **「CEX 就是骗子」**：头部合规 CEX 是行业基础设施，问题在于「信任集中」，不等于一定作恶。关键看是否合规、是否有储备金证明。

## 小结

- CEX vs DEX 的本质是「便利 vs 自主」的取舍。
- CEX 托管+订单簿+法币通道，DEX 自托管+AMM+链上透明。
- 两者长期共存且边界在融合。理解 CEX 是理解 DEX 的基础。
