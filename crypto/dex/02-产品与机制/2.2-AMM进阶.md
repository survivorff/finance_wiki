# 2.2 AMM 进阶：曲线、集中流动性与变体

> 深度：🔴 硬核
> 基础的 x*y=k 有缺陷（滑点大、资本效率低）。这篇讲为解决这些问题演化出的各种 AMM 变体。
> 前置：[2.1 AMM 是什么](./2.1-AMM是什么.md)。

## 基础 x*y=k 的两大痛点

1. **稳定币交易滑点太大**：USDC/USDT 本应 1:1，但用 x*y=k，大额兑换滑点明显，浪费。
2. **资本效率低**：流动性平摊在 0 到无穷大的所有价格上，但实际交易只发生在当前价附近，**大部分流动性「闲置」**。

针对这两点，演化出不同的 AMM。

## 一、Curve 的稳定币曲线（StableSwap）⭐

**问题**：交易两个「本该等价」的资产（USDC/USDT、ETH/stETH），希望接近 1:1、低滑点。

**方案**：Curve 用一个**混合曲线**——在「均衡点附近」像恒定和（`x+y=常数`，零滑点），在「远离均衡」时像恒定乘积（`x*y=k`，保证不被掏空）。

```
价格
 │   x*y=k(普通AMM): 中间就开始弯, 滑点大
 │  ╱
 │ ╱
 │╱___________  StableSwap: 中间一大段几乎平(低滑点)
 │            ╲   只在极端才弯(防掏空)
 └──────────────── 池子比例
```

- **效果**：在 1:1 附近提供极低滑点，是稳定币和等价资产交易之王。
- 💡 这也催生了著名的「**Curve 战争**」——各协议争夺 CRV 治理权来引导流动性激励（见 [04 生态](../04-生态与运营/README.md)）。

## 二、Uniswap V3 集中流动性（Concentrated Liquidity）⭐⭐

**问题**：V2 的流动性平摊在所有价格，资本效率低。

**方案**：LP 可以把流动性**集中在一个自选的价格区间**内。

```
V2: 流动性平摊在 [0, ∞) 所有价格   ── 大部分闲置
V3: LP 选择只在 [1900, 2100] 提供流动性 ── 资金集中在交易最频繁处
    → 同样的钱, 在该区间内提供深得多的流动性(资本效率↑ 几十倍)
```

### 代价与复杂性
- **要主动管理**：价格超出你的区间，你的流动性就「失效」了（全变成单一资产，不再赚手续费）。
- **无常损失更剧烈**：区间越窄，资本效率越高，但无常损失放大（见 [2.3](./2.3-流动性与无常损失.md)）。
- 催生了**主动 LP 管理**这个专业领域（如 Gamma、Arrakis 等「LP 管理协议」帮你自动调区间）。

> 💡 V3 把 LP 从「躺着收费」变成了「主动做市」——本质更接近 [CEX 专业做市商](../../cex/08-深水区/8.7-做市与市场微观结构.md) 的「区间报价」。资本效率和管理复杂度同时上升。

### 技术实现：tick（价格刻度）
- V3 把价格离散成「tick」，LP 在 tick 区间提供流动性。和 [CEX 撮合的 tick size](../../cex/08-深水区/8.1-撮合引擎实现细节.md) 异曲同工。数学见 [8.1](../08-深水区/8.1-AMM数学全解.md)。

## 三、Balancer 加权池（Weighted Pool）

- 推广 x*y=k 到**多种币、不等权重**：如 80% ETH / 20% USDC 的池子。
- 公式：`∏ (balance_i ^ weight_i) = k`。
- 用途：**自平衡的指数基金**（池子比例自动维持权重）、灵活的多资产池。

## 四、其他变体（了解）

| 变体 | 特点 | 代表 |
| --- | --- | --- |
| **Uniswap V4 + Hooks** | 可插件化定制 AMM 逻辑(动态费率/限价等) | Uniswap V4 |
| **DODO PMM** | 主动做市商算法, 用预言机贴近市场价 | DODO |
| **ve(3,3) 模型** | 投票托管 + 流动性激励博弈 | Curve, Aerodrome |
| **集中流动性变体** | 各链对 V3 思路的改进 | 多个协议 |

## AMM 变体对比表 ⭐

| AMM | 公式/思路 | 最适合 | 主要代价 |
| --- | --- | --- | --- |
| 恒定乘积 V2 | x*y=k | 通用、长尾币 | 滑点大、资本效率低 |
| StableSwap | 混合曲线 | 稳定币/等价资产 | 不适合波动大的币对 |
| 集中流动性 V3 | 区间内做市 | 主流币对、效率优先 | 要主动管理、无常损失放大 |
| 加权池 | 多币加权乘积 | 指数/多资产组合 | 复杂 |

## 一个核心权衡 ⭐

> 所有 AMM 创新都在权衡同一组矛盾：
> **资本效率 ↑ ←→ LP 管理复杂度 ↑ + 无常损失风险 ↑**
>
> - V2：效率低，但「存进去就不用管」，无常损失温和。
> - V3：效率高几十倍，但要主动管理区间，无常损失剧烈。
> - StableSwap：对「等价资产」效率极高，但只适用于这类资产。
>
> 没有万能的 AMM，只有「针对特定资产特性的最优曲线」。

## 小结

- 基础 x*y=k 有「稳定币滑点大、资本效率低」两大痛点，催生各种变体。
- Curve StableSwap 用混合曲线实现等价资产的极低滑点。
- Uniswap V3 集中流动性大幅提升资本效率，但要主动管理且放大无常损失。
- Balancer 加权池支持多币不等权重（自平衡指数）。
- 核心权衡：资本效率 ↑ 必然伴随管理复杂度和无常损失 ↑。
