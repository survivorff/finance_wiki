# Finance Wiki · 全球金融市场学习知识库

> 一个系统性梳理「世界上最大的几个资金市场如何搅动、流动」的个人学习知识库。
> 从底层设计原理出发，逐个赛道拆解业务模式与技术架构。

## 这个知识库想回答的核心问题

全球的钱主要沉淀在几个巨型市场里：外汇、债券、股票、大宗商品、加密货币……
每个市场都有自己的「玩家、规则、基础设施、盈利模式」。这个知识库的目标是：

1. **业务层**：每个市场里都有谁在玩？他们靠什么赚钱？资金从哪来、到哪去？
2. **技术层**：支撑这些市场运转的系统是怎么设计的？（撮合、清算、托管、风控……）
3. **原理层**：为什么是这样设计的？背后的金融逻辑、博弈与约束是什么？

学习方式：**一个赛道一个赛道来**，每个赛道都从「行业全景 → 核心业务 → 技术架构 → 盈利与博弈 → 风险与监管」逐层深入。

## 目录结构约定

每个赛道用统一的「**七维度**」结构深度剖析，方便横向对比。每个维度是一个子目录，内含多篇深度文章：

```
<赛道>/
├── README.md            # 该赛道总览 + 学习路径 + 导航
├── 00-导读与学习路径.md   # 怎么用这个知识库
├── 01-行业全景/          # 这是什么、有谁、市场多大、怎么来的
├── 02-产品体系/          # 卖什么产品、每个怎么设计
├── 03-技术架构/          # 支撑系统怎么设计（工程向）
├── 04-运营体系/          # 增长、做市、上币的台前幕后
├── 05-市场与博弈/         # 怎么赚钱、各方博弈、资金流动
├── 06-风险与监管/         # 风险地图、案例、合规、自保
├── 07-未来发展/          # 趋势与展望
└── glossary.md          # 术语表
```

> 每篇文章标注**深度等级**（🟢 入门 / 🟡 进阶 / 🔴 硬核），并提供面向不同人群（小白、从业者、工程师）的学习路径，兼顾「能看懂」和「有内幕深度」。
