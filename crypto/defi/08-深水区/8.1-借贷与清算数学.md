# 8.1 借贷与清算数学

> 深度：🔴🔴 极硬核
> 把借贷协议的利率模型、健康因子、清算用公式讲清。
> 前置：[3.1](../03-借贷协议/3.1-链上借贷怎么运转.md)、[3.2](../03-借贷协议/3.2-利率模型与清算.md)。

## 一、利用率与利率模型 ⭐

```
利用率(Utilization):
  U = 总借出 / 总存入

拐点利率模型(kinked, 见 3.2):
  当 U ≤ U_optimal(拐点, 如 80%):
    借款利率 = base_rate + (U / U_optimal) × slope1
  当 U > U_optimal:
    借款利率 = base_rate + slope1 + ((U − U_optimal)/(1 − U_optimal)) × slope2
  其中 slope2 >> slope1(拐点后陡升)

存款利率 = 借款利率 × U × (1 − 储备金率)
  → 存款人分到的, 是"借款人付的利息"按利用率和储备金折算
```

### 直觉
```
U 低: 利率低(鼓励借)
U 接近拐点: 利率温和
U 超过拐点: 利率陡升(防止池子被借空, 保护存款人能取款)
→ slope2 的陡峭是"流动性保险"
```

## 二、健康因子与清算线 ⭐

```
健康因子(Health Factor, HF):
  HF = (Σ 抵押品价值 × 清算阈值) / 总借款价值

  HF > 1: 安全
  HF = 1: 临界
  HF < 1: 可被清算!

例: 抵押 150 美元 ETH(清算阈值 80%), 借了 100 美元 USDC
  HF = (150 × 0.8) / 100 = 1.2 → 安全
  若 ETH 跌到 120 美元:
  HF = (120 × 0.8) / 100 = 0.96 < 1 → 触发清算!
  (ETH 从 150 跌到 120, 跌约 20% 就被清算)
```

## 三、清算计算 ⭐

```
清算时:
  清算人 帮还一部分借款(close factor, 如最多还 50%)
  作为回报, 清算人 以"折扣价"拿走对应抵押品(清算奖励/罚金, 如 5-10%)

例(清算奖励 8%, close factor 50%):
  借款人 HF < 1, 借了 100 USDC
  清算人还 50 USDC(50% 的债)
  拿走价值 50 × (1 + 8%) = 54 美元的 ETH 抵押品
  → 清算人净赚 4 美元(54 − 50)
  → 借款人损失了清算罚金(被折价处置抵押品)
```

### 清算的连锁数学（清算瀑布）
```
抵押品(ETH)价格 P 下跌:
  → 一批借款人 HF < 1 → 被清算 → 清算人抛售 ETH 还债
  → 抛压 → P 进一步下跌 → 更多人 HF < 1 → ...
→ 正反馈踩踏(同 CEX 连环爆仓的数学, 见 CEX 8.2)
临界: 如果 P 下跌太快, 清算来不及 或 抵押品 < 债务 → 协议坏账
```

## 四、坏账条件

```
坏账发生 当: 清算后, 抵押品价值 < 借款价值
  即清算阈值的"缓冲"被击穿(P 跌速 > 清算速度)
原因:
  - 抵押品暴跌太快(闪崩)
  - 链拥堵/Gas 高 → 清算人来不及
  - 预言机延迟/失灵(见 DEX 3.4)
缓冲: 储备金/安全模块(见 3.2) — 类似 CEX 保险基金(见 CEX 8.2)
```

## 五、和 CEX 强平数学的对照 ⭐

```
CEX 强平(见 CEX 8.2): 保证金率 < 维持保证金 → 强平
DeFi 清算: HF < 1 → 清算
本质同一个: "抵押/保证金 不足以覆盖风险 → 强制平仓"
区别:
  CEX: 维持保证金率 + 标记价 + 内部引擎执行
  DeFi: 清算阈值 + 预言机价 + 清算人机器人执行
数学结构一样, 执行者不同(中心化引擎 vs 去中心化机器人)
```

## 小结

- 利用率 U=借出/存入；拐点利率模型在 U 超过拐点后陡升（slope2），保护池子不被借空。
- 健康因子 HF=(抵押品×清算阈值)/借款，HF<1 触发清算。
- 清算：清算人还部分债、折扣拿抵押品赚清算奖励，借款人损失罚金。
- 清算瀑布（抛售→价跌→更多清算）是正反馈踩踏（同 CEX 连环爆仓数学）；坏账=清算缓冲被击穿。
- DeFi 清算和 CEX 强平数学结构相同（抵押不足→强平），区别是执行者（机器人 vs 引擎）。
