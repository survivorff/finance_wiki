# 8.1 交易所撮合系统

> 深度：🔴🔴 极硬核
> 股票撮合引擎的工程，重点讲与 [CEX 撮合](../../crypto/cex/08-深水区/8.1-撮合引擎实现细节.md) 的异同，以及集合竞价的算法。

## 连续撮合：和 CEX 几乎一样

股票的连续竞价撮合引擎，核心和 [CEX 撮合引擎](../../crypto/cex/08-深水区/8.1-撮合引擎实现细节.md) 高度一致：
```
- 订单簿: 价格档位 + FIFO 队列 + order_map(O(1)撤单)
- 撮合规则: 价格优先 + 时间优先
- 单线程内存 + 事件溯源 + 快照恢复(确定性状态机)
- 极致低延迟(同 CEX 8.6 的机制同情/绑核/避免GC)
```

> 💡 不再重复（见 [CEX 8.1](../../crypto/cex/08-深水区/8.1-撮合引擎实现细节.md) 和 [8.6](../../crypto/cex/08-深水区/8.6-低延迟与性能工程.md)）。**股票交易所和 CEX 在连续撮合的工程上殊途同归**——都是高性能确定性状态机。

## 股票撮合的独特工程点

### 1. 集合竞价算法 ⭐（CEX 没有）
开收盘的集合竞价要找「成交量最大化」的统一价（见 [3.2](../03-交易与市场结构/3.2-竞价机制.md)）。算法：

```
目标: 找一个价格 P*, 使成交量最大(并满足一系列优先级规则)
算法(简化):
  1. 收集集合竞价时段的所有买单(价格→累计买量)和卖单
  2. 对每个候选价格 P:
       可成交量 = min(在P及以上愿买的总量, 在P及以下愿卖的总量)
  3. 选可成交量最大的 P*
  4. 若多个价格成交量相同, 按"次级规则"打破平局:
       - 最小化未成交量 / 最小化价格对参考价的偏离 等
  5. 该价位所有订单按 P* 统一成交, 按时间优先分配
```

```
伪代码:
def call_auction(buy_orders, sell_orders, candidate_prices):
    best_p, best_vol = None, -1
    for p in candidate_prices:
        demand = sum(o.qty for o in buy_orders if o.price >= p)   # 愿在p买的总量
        supply = sum(o.qty for o in sell_orders if o.price <= p)  # 愿在p卖的总量
        vol = min(demand, supply)                                  # 可成交量
        if vol > best_vol or (vol == best_vol and tie_break(p) better):
            best_p, best_vol = p, vol
    return best_p   # 统一成交价
```

> 💡 集合竞价是一次性的「批量撮合」，和连续撮合（逐笔）是两种模式。这也是 [DEX CoW 批量拍卖](../../crypto/dex/02-产品与机制/2.4-聚合器与意图.md) 的原型。

### 2. 熔断与涨跌幅检查
- 撮合引擎要实时检查价格是否触发熔断/涨跌停（见 [6.3](../06-监管与风险/6.3-风险与危机案例.md)），触发则暂停或限制——这是 [CEX](../../crypto/cex/02-产品体系/2.2-合约衍生品.md) 一般没有的逻辑。

### 3. 多场所与碎片化
- 单个交易所撮合自己的订单簿，但还要处理 [NBBO、订单保护](../03-交易与市场结构/3.5-订单路由与最优执行.md)（不能在比别处差的价格成交）——跨场所协调，CEX 没有这层。

### 4. 监管报送
- 每笔成交要实时报送监管（审计追踪，如美国的 CAT），合规要求远高于 CEX。

## 性能与可靠性

```
和 CEX 一样的要求(见 CEX 8.6):
  - 低延迟(微秒级)、高吞吐、高可用、可恢复
  - 状态机复制做主备
顶级股票交易所(如 Nasdaq)的撮合系统是金融工程标杆,
许多 CEX 的撮合架构其实借鉴了传统交易所(如 LMAX 本身就是金融交易所)
```

> 💡 **历史反转**：很多人以为 CEX 的撮合技术很"新"，其实**传统交易所是撮合引擎工程的鼻祖**，
> CEX 大量借鉴了它们的设计（LMAX Disruptor 就来自传统交易所）。在撮合工程上，是加密在学股市，不是反过来。

## 小结

- 股票连续撮合引擎和 CEX 几乎一样：价格-时间优先订单簿、单线程内存、确定性状态机、低延迟。
- 独特点：集合竞价算法（找成交量最大化的统一价）、熔断检查、多场所碎片化协调、监管报送。
- 集合竞价是批量撮合，是 DEX CoW 批量拍卖的原型。
- 传统交易所是撮合工程的鼻祖，CEX 大量借鉴它（如 LMAX）——撮合技术上是加密学股市。
