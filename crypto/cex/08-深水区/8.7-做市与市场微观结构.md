# 8.7 做市与市场微观结构

> 深度：🔴🔴 极硬核
> 做市商到底怎么定价、怎么赚钱、怎么不亏？这篇讲市场微观结构（market microstructure）的核心模型。
> 前置：[4.3 做市与流动性运营](../04-运营体系/4.3-做市与流动性运营.md)、[5.3 套利](../05-市场与博弈/5.3-典型玩法与套利.md)。

## 一、做市商的基本盈亏

```
做市商在盘口两侧同时挂单:
   买价(bid) = mid − spread/2
   卖价(ask) = mid + spread/2

理想情况: 买单和卖单都成交一次 → 赚一个价差(spread)
   利润 ≈ spread × 成交量 + 交易所返佣(rebate)
```

但现实没这么美好——有两大风险吃掉利润：**库存风险**和**逆向选择**。

## 二、库存风险（Inventory Risk）⭐

```
问题: 如果市场单边下跌, 你的买单不断成交(被动接货)
      → 库存(持仓)越积越多, 且都在亏
      → 你成了"接下跌的刀"

做市商必须控制库存, 保持大致【中性】(delta neutral)
```

### 库存管理：偏移报价（Skewing）
```
持有过多多头库存 → 把报价整体下移(降低买价、降低卖价)
   → 更难买入(少接货)、更易卖出(快出货)
   → 把库存推回中性

数学上: quote_mid = fair_value − γ × inventory
   (γ 是风险厌恶系数, inventory 是当前净持仓)
```

## 三、逆向选择（Adverse Selection）⭐ 做市商最大的敌人

```
问题: 和你成交的对手, 可能是"知道得比你多的人"(知情交易者)
   - 有人突然大量吃你的卖单 → 可能他知道要涨 → 你卖低了
   - 你的挂单总是"在错误的方向被成交"

这就是逆向选择: 做市商系统性地"和聪明钱对赌且处于信息劣势"
```

### 应对
- **价差补偿**：价差要足够宽，覆盖逆向选择的预期损失（信息不对称越严重，价差越宽）。
- **快速撤改单**：感知到大单/异动立刻撤单调价（这就是为什么做市拼速度，见 [8.6](./8.6-低延迟与性能工程.md)）。
- **订单流分析**：分析订单流的「毒性（toxicity）」，识别知情交易。

> 💡 **价差的本质** = 做市商对「库存风险 + 逆向选择 + 运营成本」的补偿要求。
> 流动性好（价差小）的市场，恰恰是「逆向选择风险低、竞争充分」的市场。

## 四、Avellaneda-Stoikov 模型（学术经典）

做市定价最著名的理论框架（2008），核心给出「最优买卖报价」。

### 核心思想
```
1. 计算"保留价格(reservation price)" r:
   r = s − q × γ × σ² × (T − t)
   s=中间价, q=当前库存, γ=风险厌恶, σ=波动率, (T−t)=剩余时间
   → 库存 q 越大, 保留价越偏离中间价(促使回归中性)

2. 计算最优价差(bid-ask spread):
   spread = γ × σ² × (T−t) + (2/γ) × ln(1 + γ/k)
   → 波动率 σ 越大, 价差越宽(风险大要补偿)
   → k 反映订单到达率(流动性)

3. 报价:  bid = r − spread/2,  ask = r + spread/2
```

### 直觉解读 ⭐
- **波动率 σ↑ → 价差↑**：市场越动荡，做市越危险，价差越宽（这解释了为什么暴涨暴跌时盘口价差骤然变宽、深度变薄）。
- **库存 q↑ → 报价偏移**：持仓越偏，越要通过偏移报价拉回中性。
- **剩余时间/风险厌恶**：越接近收盘、越厌恶风险，越保守。

> 💡 你不需要会推导，但要理解结论：**价差和波动率正相关，报价随库存偏移。**
> 这解释了真实盘口的很多现象（行情剧烈时价差变宽、深度消失）。

## 五、加密做市的特殊性

| 特点 | 影响 |
| --- | --- |
| **7×24 无收盘** | 库存风险持续，没有"收盘清仓"的喘息 |
| **多所碎片化** | 同一币在多所交易，要跨所对冲库存 |
| **波动极大** | 价差天然比传统市场宽，逆向选择风险高 |
| **CEX+DEX 并存** | 可在 CEX 做市、DEX 对冲（或反之） |
| **资金费率** | 持有合约库存有资金费成本/收益, 要算进定价 |
| **项目方做市协议** | 很多是"被项目方雇佣"为其代币做市(灰色见 [4.3](../04-运营体系/4.3-做市与流动性运营.md)) |

### 跨所对冲的库存管理 ⭐
```
在 A 所被动接了多头库存(做市成交)
→ 立刻在 B 所(流动性更好处)卖出对冲, 拉回中性
→ 赚 A 所的价差/返佣, 用 B 所平掉方向性风险
这要求: 多所连接 + 低延迟 + 实时库存归集
```

## 六、AMM 做市（对照 DEX，为下一赛道铺垫）

CEX 做市是「主动报价」，DEX 的 AMM 是「被动公式定价」：

```
CEX 做市:  做市商主动挂单, 动态调价, 管理库存(本文)
AMM(DEX):  恒定乘积 x*y=k, 价格由池子比例决定, 被动成交
           LP(流动性提供者)承担"无常损失"(impermanent loss)
           ≈ 一种"永远不撤单、不调价"的做市, 用公式替代人/算法
```

> 💡 **深刻联系**：AMM 的「无常损失」本质就是「逆向选择 + 库存风险」的被动版——
> LP 总是「在错误方向被套利者成交」（价格涨时被买走涨的、跌时被塞进跌的）。
> 理解了 CEX 做市的逆向选择，你就预先理解了 DEX 的无常损失。这是两个赛道的桥梁。

## 小结

- 做市赚价差+返佣，但被「库存风险」和「逆向选择」两大敌人侵蚀利润。
- 库存风险靠偏移报价（skewing）拉回中性；逆向选择靠加宽价差+快速撤改单+订单流分析应对。
- Avellaneda-Stoikov 模型核心结论：价差与波动率正相关，报价随库存偏移。
- 加密做市特殊在 7×24、多所碎片化、波动大、跨所对冲、资金费率成本。
- AMM 的无常损失 = 逆向选择+库存风险的被动版，是通往 DEX 赛道的桥梁。
