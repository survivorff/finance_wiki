# 2.1 AMM 是什么

> 深度：🟢 入门 → 🟡 进阶 ⭐ DEX 最核心的概念
> 读懂这一篇，你就掌握了 DEX 的灵魂。

## 问题：链上为什么不能用订单簿？

回忆 [CEX 订单簿](../../cex/02-产品体系/2.1-现货交易.md)：买卖双方挂单，引擎撮合。
但搬到链上，每次挂单/撤单都要发交易、付 Gas、等出块——**又慢又贵**（EtherDelta 的失败教训，见 [1.5](../01-行业全景/1.5-发展简史.md)）。

**AMM 是另一条路：不要订单簿，不要对手方，用「公式 + 资金池」自动定价。**

## 核心机制：恒定乘积公式 x*y=k ⭐

### 一个池子，两种币
想象一个资金池，里面有两种币，比如 ETH 和 USDC：
```
池子里:  x 个 ETH,  y 个 USDC
规则:   x × y = k  (k 恒定不变)
价格:   1 个 ETH 的价格 = y / x  (池里 USDC 数量 / ETH 数量)
```

### 交易如何发生
你想用 USDC 买 ETH：
```
你往池子里【加入 USDC】(y 变大) → 从池子里【取出 ETH】(x 变小)
但必须保持 x × y = k 不变
→ 所以你加的 USDC 越多, 能取的 ETH 边际越少(价格越来越贵)
```

### 一个具体算例 ⭐
```
初始池: 10 ETH, 20000 USDC   →  k = 10 × 20000 = 200000
        当前 ETH 价格 = 20000/10 = 2000 USDC

你要用 USDC 买 1 个 ETH:
  买后池里剩 9 ETH, 为保持 k=200000:
  新的 USDC 数量 = 200000 / 9 ≈ 22222
  → 你需要付 22222 − 20000 = 2222 USDC 买这 1 个 ETH
  → 均价 2222, 而不是 2000!
```

> ⭐ **这就是滑点（Slippage）**：你买得越多，价格越偏离初始价。
> 因为公式是条曲线，你「沿着曲线移动」，买得越多走得越远，价格越差。

## 关键概念

### 1. 流动性池（Liquidity Pool）
- 池子里的两种币就是「流动性」。池子越大（钱越多），同样的交易引起的价格变动越小（滑点越小）。
- **池子深度 = CEX 的盘口深度**（[对照 CEX](../../cex/02-产品体系/2.1-现货交易.md)）。

### 2. 流动性提供者（LP, Liquidity Provider）⭐
- **任何人**都可以把两种币按比例存进池子，成为 LP。
- LP 赚取**这个池子所有交易的手续费**（如每笔 0.3% 分给 LP）。
- 💡 **这是 AMM 的革命性所在**：CEX 做市是专业机构的特权，AMM 让**人人都能当做市商**赚手续费。
- 代价：LP 要承担「无常损失」（见 [2.3](./2.3-流动性与无常损失.md)）。

### 3. 滑点与价格影响
```
滑点 ∝ 你的交易量 / 池子深度
小池子里买大单 → 滑点巨大(可能买在很差的价格)
所以交易前要设"滑点容忍度"(超过就不成交, 防止被坑)
```

### 4. 套利者维持价格正确 ⭐
```
如果池子里 ETH 价格(2000) 低于外部市场(2100):
  套利者来池子买便宜的 ETH → x 减少 → 池子 ETH 价格上升
  直到池子价格 ≈ 外部市场价
```
- 💡 **套利者是 AMM 的「价格校准器」**。AMM 自己不知道「真实价格」，是套利者通过交易把池子价格拉到和市场一致。
- 这也意味着：套利者赚的钱，部分来自 LP（这是无常损失的来源之一，见 [2.3](./2.3-流动性与无常损失.md)）。

## AMM vs 订单簿（核心对照）⭐

| | 订单簿（CEX） | AMM（DEX） |
| --- | --- | --- |
| 定价 | 买卖挂单撮合 | 数学公式 x*y=k |
| 谁提供流动性 | 专业做市商挂单 | 人人可存池当 LP |
| 对手方 | 需要另一个交易者 | 和池子交易，不需对手 |
| 流动性 | 可能某价位没挂单 | 任何价位都能成交(只是滑点) |
| 复杂度 | 撮合引擎复杂 | 合约极简(一个公式) |
| 价格来源 | 内生(成交决定) | 外部套利者校准 |

## 为什么 x*y=k 如此巧妙 💡

```
1. 永远有流动性: 无论价格多高多低, 曲线上总有对应点能成交
2. 自动定价: 不需要任何人报价, 公式自动给出价格
3. 无需对手方: 你和池子交易, 不用等另一个人
4. 合约极简: 核心逻辑就一个乘法恒等式, 安全且 Gas 省
5. 无需许可: 任何人可创建任意币对的池子、可当 LP
```

> 这个简单的公式，用最小的代码解决了「链上做市」这个看似无解的难题。这是 DeFi 最优雅的设计之一。

## ⚠️ 小白须知

- **滑点**：交易大单/小池子时，实际成交价可能远差于显示价。务必看「预计获得」和设滑点保护。
- **无常损失**：当 LP 不是稳赚，可能不如直接拿着币（见 [2.3](./2.3-流动性与无常损失.md)）。
- **任何人能建池**：所以有大量骗局池（貔貅、仿冒币），见 [06 安全](../06-风险与安全/README.md)。

## 小结

- AMM 用「恒定乘积 x*y=k + 资金池」替代订单簿，自动定价、无需对手方。
- 价格 = 池中两币的比例；交易沿曲线移动产生滑点。
- LP（人人可当）提供流动性赚手续费，是 AMM 的革命性创新。
- 套利者通过交易把池子价格校准到外部市场价。
- 这个简单公式优雅地解决了「链上做市」难题。
