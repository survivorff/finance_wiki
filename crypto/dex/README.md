# 加密货币 · DEX（去中心化交易所）深度知识库

> Decentralized Exchange，去中心化交易所。
> 代表：Uniswap、Curve、PancakeSwap、Hyperliquid、dYdX、Jupiter、Aerodrome 等。

如果说 [CEX](../cex/README.md) 是「把钱交给一家公司换效率」，那 DEX 就是另一种答案：
**用代码和密码学，让陌生人之间无需信任任何中介，就能交易资产。**

DEX 是对 CEX「中心化托管原罪」的根本性回应。理解 DEX，你会看到金融最激进的一次重构——
**没有公司、没有客服、没有托管、没有 KYC，只有部署在区块链上、任何人可验证的智能合约。**

这个知识库的目标：让**小白**理解「去中心化交易到底怎么实现」，让**从业者/工程师**挖到 AMM 数学、MEV、合约安全、L2 等深水细节。

## 🎯 读完应该建立的「认知锚点」⭐

1. **DEX 的灵魂是「自托管 + 链上可验证」**：你的币始终在自己钱包，交易由公开的智能合约执行，人人可审计。这是对 CEX 黑盒的根本性回答。
2. **AMM（自动做市商）是 DEX 最伟大的发明**：用一个数学公式 `x*y=k` 和一个资金池，取代了订单簿和做市商——任何人都能当做市商。
3. **没有免费的午餐**：DEX 用「无常损失」「MEV 被夹」「Gas 费」「合约风险」换来了去中心化。
4. **MEV 是链上世界独有的「隐形税」**：你的交易在打包前是公开的，机器人可以抢跑、夹击你。
5. **DEX 和 CEX 在解决同样的问题**（撮合、流动性、清算、价格发现），只是范式完全不同。**对照学习价值极高。**
6. **DEX 把信任从「机构」转移到了「代码」**——但代码也会有 bug，信任风险没消失，只是换了形态。

## 一张图看懂 DEX 与 CEX 的根本区别

```
   CEX(中心化)                          DEX(去中心化)
┌──────────────────┐              ┌──────────────────────┐
│ 你 → 把币存入交易所 │              │ 你 → 币始终在自己钱包    │
│      (失去控制权)   │              │      (自托管, 私钥自持)  │
│         ↓         │              │         ↓             │
│   交易所数据库记账   │              │   你的钱包 ↔ 智能合约    │
│   (黑盒, 内部撮合)  │              │   (链上公开, 人人可验证) │
│         ↓         │              │         ↓             │
│   你看到一个余额数字 │              │   链上真实转账, 可查    │
│   = 交易所欠你的债权 │              │   = 你真正拥有的资产    │
└──────────────────┘              └──────────────────────┘
   信任"公司"                          信任"代码"
```

> **关键洞察**：CEX 要你信任一家公司不作恶；DEX 让你不必信任任何人，只需信任（可审计的）代码。
> 信任从「人/机构」转移到了「数学/代码」——这是 DEX 的全部哲学，也是它全部风险的来源（代码有 bug 怎么办？）。
