# 3.2 智能合约与 DEX 架构

> 深度：🔴 硬核
> 一个 AMM DEX 由哪些合约组成？它们怎么协作？这篇拆解 Uniswap 式架构。
> 前置：[2.1 AMM](../02-产品与机制/2.1-AMM是什么.md)、[3.1 区块链基础](./3.1-区块链基础.md)。

## Uniswap V2 式架构（经典三件套）⭐

```
┌──────────────┐    创建池子     ┌──────────────┐
│ Factory 工厂  │ ───────────────▶│ Pair 池子合约  │ (每个交易对一个)
│ (部署/登记池) │                 │ 持有储备 + 撮合 │
└──────────────┘                 └──────────────┘
                                        ▲
┌──────────────┐    路由调用            │
│ Router 路由   │ ──────────────────────┘
│ (用户交互入口)│  拆分路径、多跳、滑点保护
└──────────────┘
      ▲
   用户钱包
```

### 三个核心合约
| 合约 | 职责 |
| --- | --- |
| **Factory（工厂）** | 创建并登记每个交易对的池子合约；记录所有池子 |
| **Pair/Pool（池子）** | 持有两种币的储备、执行 x*y=k 兑换、记账 LP 份额 |
| **Router（路由）** | 用户实际交互的入口；处理多跳路径、滑点保护、封装常用操作 |

### 一笔 swap 在合约里发生了什么
```
1. 用户(经前端)调用 Router.swapExactTokensForTokens(...)
2. Router 计算路径(可能 A→B→C 多跳), 校验滑点(minAmountOut)
3. Router 调用对应 Pair 合约
4. Pair: 收到输入币, 按 x*y=k 算出输出币, 转给用户, 更新储备
5. 校验: 实际到手 ≥ minAmountOut, 否则整笔回滚(链上原子性)
```

## LP 份额：流动性凭证（LP Token）

```
你存入两种币当 LP → 池子铸造 "LP Token" 给你(代表你占池子的份额)
赎回时 → 销毁 LP Token, 按份额取回两种币 + 累积的手续费
```
- LP Token 本身也是个代币，可以转账、抵押、再质押（可组合性，见 [1.4](../01-行业全景/1.4-DeFi生态位.md)）。
- 手续费机制：每笔交易留一点（如 0.3%）在池子里，使 LP 赎回时取回的币变多。

## 「协议」与「前端」分离 ⭐ 去中心化的关键

```
Uniswap "协议" = 部署在链上的合约(Factory/Pair/Router) ── 去中心化、不可篡改、永久运行
uniswap.org "前端" = 一个网页 ── 中心化、可被下架/封锁

关键: 任何人都能写自己的前端 或 直接调用合约, 访问同一套协议
     → 即使官网被封, 协议依然在链上运行, 你仍能用
```

- 💡 这是 DEX「抗审查」的核心：**前端可以被封，但链上协议封不掉。**
- ⚠️ 但现实中：前端常做合规过滤（屏蔽某些地区/代币）、预言机/治理仍可能有中心化环节——「去中心化」是程度问题，不是绝对的。

## 合约的可升级性问题 💡

```
不可升级(如 Uniswap 核心): 部署后永不改 → 最可信, 但有 bug 也改不了, 只能发新版本(V2→V3)
可升级(代理合约 proxy):   能改逻辑修 bug → 灵活, 但"谁能升级"就是个中心化/信任点
                          (升级权限若被盗, 整个协议可被改成恶意)
```

- 这是个根本权衡：**不可篡改的可信 vs 可修复的灵活**。
- 很多协议用「多签 + 时间锁（timelock）」控制升级权限：改动要多签批准且延迟生效，给用户反应时间。

## 工厂模式的意义

- 任何人都能通过 Factory **无许可地创建任意币对的池子**——这是「人人可上币、可做市」的技术基础。
- 也是骗局币泛滥的技术原因（谁都能建池子，包括貔貅盘，见 [06](../06-风险与安全/README.md)）。

## 与 CEX 架构的根本对照 ⭐

| | CEX（[对照](../../cex/03-技术架构/3.1-整体架构总览.md)） | DEX |
| --- | --- | --- |
| 撮合 | 中心化内存撮合引擎 | 池子合约里的公式 |
| 账本 | 中心化数据库 | 链上储备 + LP Token |
| 部署 | 自己的服务器集群 | 公链上的合约 |
| 谁运行 | 交易所公司 | 全网验证者执行合约 |
| 改代码 | 随时发版 | 不可升级 或 多签+时间锁 |
| 上币 | 上币团队审批 | 工厂合约无许可创建 |

## 小结

- 经典 AMM DEX = Factory（建池）+ Pair/Pool（储备与兑换）+ Router（用户入口、多跳、滑点保护）三件套。
- LP 份额用 LP Token 表示，可在 DeFi 里再组合。
- 「协议（链上合约）vs 前端（网页）」分离是抗审查的核心，但去中心化是程度问题。
- 合约可升级性是「不可篡改的可信 vs 可修 bug 的灵活」的根本权衡，常用多签+时间锁折中。
- 工厂模式让人人可无许可建池，是开放性的来源，也是骗局币的温床。
