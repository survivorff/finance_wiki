# 3.1 订单簿与撮合

> 深度：🟡 进阶
> 股票撮合的内核和 [CEX 撮合](../../crypto/cex/03-技术架构/3.2-撮合引擎.md) 几乎一样。已读过 CEX 的可快速过。

## 订单簿（Order Book）

和 [CEX 现货](../../crypto/cex/02-产品体系/2.1-现货交易.md) 完全一样的概念：

```
卖单 asks (从低到高)
  150.05  | ████ 500股
  150.03  | ██ 200股
  150.01  | ███ 300股      ← 最优卖价
──────────────────────── 价差 spread
  149.99  | ████ 400股     ← 最优买价
  149.97  | ██ 200股
买单 bids (从高到低)
```

- **最优买价/卖价、价差、深度**——概念和 CEX 一致。
- 股票的最小价格变动单位叫 **tick size**（如 1 美分）。

## 撮合规则：价格优先 + 时间优先 ⭐

和 [CEX 撮合引擎](../../crypto/cex/08-深水区/8.1-撮合引擎实现细节.md) 完全相同：
```
1. 价格优先: 买价高的先成交, 卖价低的先成交
2. 时间优先: 同价位, 先挂的先成交(FIFO)
```

- 💡 这套「价格-时间优先」是全球交易所的通用规则，股市和 CEX 共享同一套撮合内核。
- 工程实现也类似（价格档位 + FIFO 队列），见 [CEX 8.1](../../crypto/cex/08-深水区/8.1-撮合引擎实现细节.md)。

## 股市撮合的独特之处

### 1. 不是 7×24，有开盘收盘
- 股市有**交易时段**（如 9:30-16:00），有开盘前/收盘后的集合竞价（见 [3.2](./3.2-竞价机制.md)）。
- 对照 [CEX 7×24 连续交易](../../crypto/cex/03-技术架构/3.7-高可用与安全工程.md)——股市的「收盘」给了市场消化信息、公司发公告的窗口。

### 2. 涨跌幅限制 / 熔断
- 很多市场有涨跌停板（如 A 股 ±10%）或熔断机制（见 [1.4](../01-行业全景/1.4-发展简史.md)），限制单日波动。
- 对照加密：[CEX 无涨跌停](../../crypto/cex/02-产品体系/2.2-合约衍生品.md)，可瞬间暴涨暴跌——股市用「限制」换稳定。

### 3. 多个交易所并存 + 碎片化
- 同一只股票可能在多个交易所、多个暗池交易（见 [3.4](./3.4-暗池与场外.md)、[3.5](./3.5-订单路由与最优执行.md)）。
- 这是股市比 CEX 复杂的地方——CEX 通常是单一平台内的统一订单簿。

## 撮合的几个细节

```
- 连续竞价时段: 来一单撮一单(像 CEX)
- 集合竞价时段(开收盘): 集中撮合, 统一定价(见 3.2)
- 做市商报价也进订单簿, 提供流动性(见 2.2)
```

## 小结

- 股票订单簿和撮合规则（价格-时间优先）与 CEX 完全相同，共享同一套内核。
- 股市独特处：有开收盘（非 7×24）、有涨跌幅/熔断限制、多交易所碎片化。
- 这些差异都指向股市的取向：用「时段、限制、分散」换取稳定和秩序。
