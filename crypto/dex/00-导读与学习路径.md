# 00 · 导读与学习路径

> 在深入前，花 5 分钟理解「这个知识库怎么用」，以及 DEX 和 CEX 的核心对照。

## 这个知识库为谁而写

- **小白**：用过 CEX，但没碰过链上钱包/DEX，想搞懂「去中心化交易到底怎么回事」。
- **DeFi 从业者/投研**：想系统理解 AMM、流动性、代币经济、治理。
- **智能合约工程师**：想理解 AMM 数学、合约实现、MEV、安全。
- **想对照学习的人**：已读完 CEX，想看「同一个问题的另一种答案」。

## 阅读约定（和 CEX 一致）

- **🟢🟡🔴** 标注深度，按需取舍。

- **💡 内幕/洞察**，**⚠️ 误区/风险**。

## ⭐ DEX vs CEX 总对照表（先建立全局认知）

| 维度 | CEX | DEX |
| --- | --- | --- |
| **资产保管** | 交易所托管 | 用户自托管（私钥自持） |
| **撮合/定价** | 中心化订单簿撮合引擎 | 智能合约（AMM 公式 / 链上订单簿） |
| **做市** | 专业做市商挂单 | 任何人存入资金池当 LP（AMM） |
| **速度** | 微秒级（内存） | 秒级+（受出块限制） |
| **成本** | 手续费 | 手续费 + Gas + 滑点 + MEV 损耗 |
| **法币通道** | 有 | 一般没有（需第三方/CEX 中转） |
| **KYC** | 通常强制 | 无（匿名、抗审查） |
| **透明度** | 黑盒 | 全链上公开可验证 |
| **资产范围** | 精选币 | 任何人可上任何币（鱼龙混杂） |
| **客服/兜底** | 有客服、可能赔付 | 无客服、操作不可逆、无人兜底 |
| **核心风险** | 平台跑路/挪用/被黑 | 合约漏洞/貔貅/无常损失/MEV/私钥丢失 |
| **信任对象** | 信任公司不作恶 | 信任代码无 bug |
| **代表** | Binance, Coinbase, OKX | Uniswap, Curve, Hyperliquid |

> 💡 把这张表记住，后面每一篇都是在某一格里深挖。**DEX 不是「更好的 CEX」，而是「用不同范式解决同样问题，各有取舍」。**

## 一个重要的心态

> DEX 世界**没有客服、没有撤销、没有兜底**。
> 转错地址、签错授权、踩中貔貅盘、私钥泄露——**钱就是真的没了，没有任何人能帮你找回。**
> 去中心化的「自由」与「自我负责」是一体两面。学习 DEX，第一课就是敬畏这种「不可逆」。
