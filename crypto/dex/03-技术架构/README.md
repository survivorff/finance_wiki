# 03 · 技术架构

> 深度：🔴 硬核为主
> 目标：理解 DEX 的技术底层——智能合约、链上撮合、预言机、跨链、钱包基础设施。

DEX 的「技术架构」和 [CEX 的技术架构](../../cex/03-技术架构/README.md) 是完全不同的范式：
CEX 是「自建的中心化服务器集群」，DEX 是「部署在公链上的智能合约 + 链下辅助设施」。

## DEX 技术栈鸟瞰

```
┌─────────────────────────────────────────────────────────┐
│ 前端层    DApp 网站 / 钱包(MetaMask等)                      │
├─────────────────────────────────────────────────────────┤
│ 接入层    RPC 节点(Infura/Alchemy) · 索引器(The Graph)     │
├─────────────────────────────────────────────────────────┤
│ 合约层    DEX 智能合约(AMM/路由/工厂) ← 核心, 链上          │
├─────────────────────────────────────────────────────────┤
│ 公链层    以太坊/L2/Solana... (执行+共识+结算)             │
├─────────────────────────────────────────────────────────┤
│ 辅助层    预言机(Chainlink) · 跨链桥 · MEV 基础设施         │
└─────────────────────────────────────────────────────────┘
```

## 几个核心工程主题

| 主题 | DEX 的答案 | 对照 CEX |
| --- | --- | --- |
| 撮合在哪 | 链上合约/专用链 | 中心化内存引擎 |
| 状态存哪 | 区块链(全网共识) | 中心化数据库 |
| 价格从哪来 | AMM 公式 / 预言机 | 内部撮合成交 |
| 谁来执行 | 矿工/验证者打包交易 | 自己的服务器 |
| 怎么保证安全 | 合约审计 + 链的共识安全 | 内部风控 + 钱包安全 |
