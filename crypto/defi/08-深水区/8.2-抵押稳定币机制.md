# 8.2 抵押稳定币机制（CDP）

> 深度：🔴🔴 极硬核
> 超额抵押稳定币（如 DAI）怎么运转——CDP、抵押率、稳定费、清算拍卖。
> 前置：[2.2 超额抵押稳定币](../02-稳定币/2.2-法币与超额抵押稳定币.md)、[8.1](./8.1-借贷与清算数学.md)。

## CDP：抵押债仓 ⭐

```
CDP(Collateralized Debt Position, 抵押债仓): 超额抵押稳定币的核心机制
  你开一个"金库(vault)":
    存入抵押品(ETH) → 生成(借出)稳定币(DAI)
    本质: 你用 ETH 做抵押, "借出"了 DAI(等于自己铸造了 DAI)
  关闭金库:
    还回 DAI + 稳定费(利息) → 取回 ETH 抵押品
→ DAI 的发行 = 用户超额抵押"借"出来的(不是凭空印)
```

## 关键参数 ⭐

```
抵押率(Collateralization Ratio):
  = 抵押品价值 / 借出 DAI 价值
  例: 抵押 150 美元 ETH, 借 100 DAI → 抵押率 150%
最低抵押率(清算线): 如 150% — 跌破就清算
稳定费(Stability Fee): 借 DAI 要付的"利息"(年化%) — 类似借贷利率
  → 调节稳定费可影响 DAI 的供应和锚定(费高→少借→DAI 减少→价格升)
```

## 锚定维持机制 ⭐

DAI 怎么维持 ≈ 1 美元？多重机制：

```
1. 套利(供需): DAI > 1 → 划算去抵押铸 DAI 卖出 → 供应增→价格回落
              DAI < 1 → 划算买便宜 DAI 还债赎回抵押品 → 供应减→价格回升
2. 稳定费调节: 调利息影响借 DAI 的意愿 → 调节供应
3. 清算保证: 每个 DAI 始终有超额抵押支撑 → 底气
4. 锚定稳定模块(PSM): 允许用 USDC 等 1:1 兑换 DAI → 强力锚定
   (但引入了对中心化稳定币的依赖, 见 2.2 的"去中心化打折")
```

## 清算拍卖 ⭐

```
抵押品跌破最低抵押率 → 清算:
  方式: 把抵押品"拍卖"掉来还债 + 罚金
  拍卖类型:
    英式拍卖(竞价): 清算人竞价买抵押品
    荷兰式拍卖(降价): 价格从高往低降, 第一个接受的成交(更快, 防拥堵)
  → 拍卖所得还掉 DAI 债务 + 清算罚金, 剩余还给原用户
极端情况: 抵押品暴跌+拍卖不顺 → 系统坏账 → 用"协议盈余"或增发治理代币兜底
```

- 💡 **黑色星期四（2020.3）教训**：ETH 暴跌 + 以太坊拥堵 → DAI 清算拍卖中有人以接近 0 的价格拍走抵押品 → 系统产生坏账 → MakerDAO 靠增发治理代币(MKR)拍卖来补窟窿。**链拥堵让清算机制失灵**（同 [8.1 坏账条件](./8.1-借贷与清算数学.md)）。

## 多抵押品与风险参数 💡

```
现代 CDP 稳定币支持多种抵押品(ETH、wBTC、LP 代币、甚至 RWA):
  每种抵押品有自己的风险参数:
    抵押率(波动大的资产要求更高抵押率)
    债务上限(限制单一抵押品的风险敞口)
    清算罚金
→ 治理(DAO)调节这些参数管理系统风险
→ 引入 RWA(见 07)做抵押 = 把现实资产风险引入(收益更稳但更中心化)
```

## CDP vs 借贷协议 vs CCP 💡

```
CDP(铸稳定币): 抵押 ETH "铸造" DAI(创造新稳定币)
借贷协议(见 03): 抵押 ETH "借出"池子里已有的 USDC(不创造新币)
→ 区别: CDP 是"凭抵押铸币", 借贷是"从池子借现有币"
共同: 都靠超额抵押 + 清算
对照 CCP(见 股票 4.2): 都用"抵押/保证金 + 违约处置"管理对手方风险
→ 超额抵押 + 清算/拍卖 是 DeFi 版的"保证金 + 违约瀑布"
```

## 小结

- CDP（抵押债仓）：存抵押品（ETH）铸造稳定币（DAI），还币+稳定费取回抵押品——DAI 是超额抵押"借"出来的。
- 关键参数：抵押率（如 150%）、最低抵押率（清算线）、稳定费（利息，可调节供应/锚定）。
- 锚定靠套利+稳定费调节+清算保证+PSM（1:1 兑 USDC，但引入中心化依赖）。
- 清算用拍卖（英式/荷兰式）处置抵押品；黑色星期四教训：链拥堵让清算失灵产生坏账（靠增发 MKR 补）。
- CDP（凭抵押铸币）vs 借贷（从池子借现有币），都靠超额抵押+清算，是 DeFi 版的"保证金+违约瀑布"。
