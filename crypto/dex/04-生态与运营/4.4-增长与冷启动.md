# 4.4 增长与冷启动

> 深度：🟡💡
> DEX 没有市场部、没有 BD，怎么获客和增长？靠的是一套加密原生的「代币激励 + 预期」打法。

## DEX 增长的核心逻辑

```
CEX 增长(对照): 烧钱买量、返佣、KOL、赞助(公司花钱) — 见 CEX 4.1
DEX 增长:      用"代币"和"代币预期"激励行为(协议印钱)
              → 把营销预算换成了代币分发
```

## 一、空投预期驱动的增长 ⭐

```
打法: 协议(暂)不发币, 但暗示"早期用户未来可能空投"
     → 用户为博空投, 提前来交易、提供流动性、做任务
     → 协议零成本获得大量早期用户和流动性(TVL/交易量)
     → 真发币空投时, 这些数据又成了宣传资本
```

- **撸毛党（空投猎人）**：专门为博空投而在各新协议刷交互的群体，规模庞大。
- ⚠️ **女巫攻击（Sybil）**：一个人用成百上千个钱包伪装成多个用户骗空投 → 协议要做「反女巫」识别。
- 💡 这是 DeFi/加密独有的增长黑客手段——**用「未来代币的预期」驱动「现在的真实使用」**，几乎零现金成本。

## 二、积分系统（Points）— 空投的升级版

```
近年流行: 协议明确推出"积分", 用户行为攒积分, 暗示积分将来兑换代币空投
  → 比模糊的"空投预期"更明确、更可控
  → 协议能用积分规则精准引导用户做想要的行为(存款/交易/拉新)
  → 代表: 各大新协议的 points 活动
```

- 优点：可控、可调、延迟发币（给协议更多灵活性）。
- ⚠️ 争议：积分规则不透明、可随时更改、最终兑换比例由协议说了算——用户承担「画饼」风险。

## 三、吸血鬼攻击（Vampire Attack）⭐💡

```
经典打法: 新协议直接"复制"老协议, 再用代币激励把老协议的流动性/用户"吸"过来
案例: SushiSwap vs Uniswap(2020, 见 1.5)
  SushiSwap 复制 Uniswap 代码 + 发 SUSHI 激励
  → 引导用户把 Uniswap 的 LP 份额迁移到 Sushi(给 SUSHI 奖励)
  → 短期内"吸走"了 Uniswap 大量流动性
  → 逼得 Uniswap 发 UNI 反击
```

- 💡 **吸血鬼攻击只在 DeFi 可能**：因为协议**开源、可分叉、可组合**——你能直接复制对手的代码，再用代币激励抢用户。
- 这是「无许可创新」的双刃剑：创新极快，但护城河也极脆弱（代码随时被抄）。

## 四、可组合性带来的「自然增长」

```
被集成 = 增长: 你的 DEX 被聚合器(1inch/Jupiter)路由、被其他协议调用
            → 自动获得流量, 无需自己拉客
→ 成为"被依赖的基础设施"是 DEX 最强的护城河(如 Uniswap)
```

- 对照 [CEX 生态飞轮](../../cex/02-产品体系/2.6-生态产品.md)：DEX 的飞轮是「被组合 → 流动性深 → 更多协议来组合」。

## 五、DEX 的护城河（为什么 Uniswap 难被颠覆）💡

代码能抄、流动性能被吸，那头部 DEX 凭什么稳？

| 护城河 | 说明 |
| --- | --- |
| **流动性深度** | 飞轮效应，深流动性自我强化（同 CEX） |
| **品牌与信任** | 老协议、被审计、没出过大事 = 信任 |
| **被集成（网络效应）** | 被无数协议/聚合器依赖，成为基础设施 |
| **持续创新** | V2→V3→V4，不断领先 |
| **多链部署** | 抢占各链生态位 |

> 💡 **关键反差**：DEX 的代码可以被复制（吸血鬼攻击），但「流动性 + 信任 + 被集成」复制不了。
> SushiSwap 抄得走代码，抄不走 Uniswap 多年沉淀的信任和生态地位。这就是为什么 Uniswap 至今仍是龙头。

## 小结

- DEX 用「代币激励 + 代币预期」代替 CEX 的现金营销，几乎零现金成本。
- 空投预期驱动早期增长（催生撸毛党和女巫攻击）；积分系统是其可控升级版。
- 吸血鬼攻击（复制代码+代币激励抢流动性）只在开源可组合的 DeFi 可能，是双刃剑。
- 被聚合器/协议集成是 DEX 的自然增长和最强护城河。
- 代码可抄，但流动性、信任、被集成的网络效应抄不走——这是头部 DEX 的护城河。
