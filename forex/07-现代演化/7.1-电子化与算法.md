# 7.1 电子化与算法

> 深度：🟡 进阶
> FX 从电话喊价到算法主导的演变。和股票 7.2、CEX 低延迟工程同源。

## 电子化历程

```
过去: 交易员电话/聊天议价(人工, 见 3.2)
现在: 几乎全电子
  - 电子平台(EBS/Reuters/各 ECN)聚合报价
  - 算法做市(银行用算法自动报价对冲)
  - 算法执行(机构用 TWAP/VWAP 拆单, 同 股票 8.3)
  - 高频交易(延迟套利、统计套利, 同 股票 7.2)
```

## 算法交易的主导

```
FX 算法交易类型(和股票/加密高度重合):
  做市算法: 自动双边报价
  执行算法: 把大单拆开(VWAP/TWAP/IS, 见 股票 8.3)
  套利算法: 跨平台/三角套利(同 CEX 套利, 见 CEX 5.3)
  趋势/统计算法: 量化策略
```

## 速度竞赛 💡

```
FX 高频同样追逐低延迟(同 股票 7.2、CEX 8.6):
  - 服务器托管在交易平台机房旁
  - 低延迟网络、专用硬件
  - "最后观望"(Last Look, 见 3.2)就是延迟博弈的产物
→ "快的人收割慢的人"在 FX 同样上演
```

## 电子化的影响

```
正面: 点差更小、效率更高、价格更透明(对大玩家)
负面: 
  - 散户更难(对手是算法)
  - 流动性"幽灵化"(算法在波动时瞬间撤单 → 流动性骤降)
  - 闪崩风险(算法连锁反应, 同 股票闪电崩盘)
```

- 💡 2019 年等曾发生 FX 闪崩（某货币对几分钟内剧烈波动），和 [股票闪电崩盘](../../stocks/06-监管与风险/6.3-风险与危机案例.md) 同源——**电子化/算法化市场的共同脆弱性**。

## 小结

- FX 几乎全电子化，算法主导（做市/执行/套利/统计），和股票/加密的算法交易高度重合。
- 同样的低延迟速度竞赛（托管/专用硬件），"最后观望"是延迟博弈产物。
- 电子化让点差更小但散户更难、流动性"幽灵化"、有闪崩风险（同股票闪电崩盘）。
