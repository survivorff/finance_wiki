# 6.1 风险地图

> 深度：🟡 进阶
> CEX 的风险可分为四大类：资产安全、信用/挪用、市场、合规/法律。逐一拆解。

## 风险全景

```
┌──────────────── CEX 风险地图 ────────────────┐
│                                               │
│ 1. 资产安全风险   被黑、被盗、私钥失守           │
│ 2. 信用/挪用风险  挪用用户资产、跑路 ⭐最致命     │
│ 3. 市场风险      流动性枯竭、插针、连环爆仓、宕机 │
│ 4. 合规/法律风险  无牌、被处罚、通道被切          │
│                                               │
└───────────────────────────────────────────────┘
```

## 1. 资产安全风险（被黑、被盗）

- **黑客攻击**：热钱包被盗、私钥泄露、签名流程被欺骗、内部作恶。
- **历史案例**：Mt.Gox（~85万 BTC）、Coincheck（~5亿美元 NEM）、Bybit（2025，~15亿美元级）。
- **技术防线**：冷热钱包分离、多签/MPC、HSM、提币白名单、签名验证（见 [3.3](../03-技术架构/3.3-钱包与充提.md)）。
- 💡 演进：攻击从「黑热钱包」转向「欺骗签名流程」，盲签是大忌。

## 2. 信用 / 挪用风险（最致命）⭐

- 交易所**挪用用户托管资产**去放贷、自营、还债，投资失败 + 挤兑 → 资不抵债 → 跑路/破产。
- **本质**：「Not your keys, not your coins」——你的币在交易所，你拥有的只是**债权**，不是币。一旦交易所资不抵债，你的债权可能血本无归。
- **案例**：FTX（挪用）、Mt.Gox、Celsius/Voyager（理财暴雷）。
- **防线**：客户资产隔离（监管强制）、储备金证明（PoR）、第三方审计。
- ⚠️ 这是 CEX **独有的、最大的、最难防的风险**——因为它发生在你看不见的黑盒里。

## 3. 市场风险

| 风险 | 说明 |
| --- | --- |
| **流动性枯竭** | 极端行情下深度骤降，无法平仓或滑点巨大 |
| **插针/操纵** | 薄深度市场被砸出瞬间极值，触发连环强平(见 [5.4](../05-市场与博弈/5.4-市场操纵与黑幕.md)) |
| **连环爆仓** | 高杠杆下 跌→强平→砸盘→更跌→更多强平 的踩踏 |
| **系统宕机** | 极端行情时卡顿/宕机，"该跌时登不上"，无法操作致损 |
| **资金费率剧变** | 合约持仓成本骤增 |

> ⚠️ 市场风险中，**高杠杆是最大的风险放大器**。100x 杠杆下 1% 波动就爆仓，是散户亏损的头号原因。

## 4. 合规 / 法律风险

- 无牌经营、违反 AML/KYC、被监管处罚甚至高管被诉。
- **案例**：Binance 与美国和解（43亿美元，CZ 卸任）；BitMEX 高管被诉；多个所被各国封禁。
- **连带风险**：交易所被监管打击 → 法币通道被切 → 用户出入金困难甚至资产被冻。
- 💡 对用户的影响：你用的交易所如果被你所在国封禁，可能突然无法访问、无法出金。

## 风险的「可见性」分级 💡

理解风险，关键是知道哪些你能看到、哪些看不到：

| 风险 | 可见性 | 你能做什么 |
| --- | --- | --- |
| 市场风险(波动/爆仓) | 高（自己操作导致） | 控制杠杆、设止损、不赌 |
| 资产安全(被盗) | 中（看历史和声誉） | 选安全记录好的所、开启全部安全设置 |
| 合规风险 | 中（看牌照和新闻） | 选合规所、关注监管动态 |
| 信用/挪用风险 | **极低（黑盒）** | 选有 PoR/审计的所，**分散存放，大额自托管** |

> ⭐ **最致命的挪用风险恰恰最不可见**。这就是为什么「不要把大额资产长期放在交易所」是行业铁律。

## 小结

- 四大风险：资产安全（被盗）、信用挪用（最致命）、市场（波动/爆仓/宕机）、合规法律。
- 挪用风险是 CEX 独有且最难防的，因为发生在不可见的黑盒里。
- 市场风险中高杠杆是最大放大器。
- 风险管理的关键是分清「可见 vs 不可见」，对不可见的挪用风险，最好的防御是分散 + 自托管大额。
