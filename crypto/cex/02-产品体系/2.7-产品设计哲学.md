# 2.7 产品设计哲学

> 深度：🟡💡 内幕视角
> 跳出单个产品，看 CEX 整个产品矩阵背后的设计逻辑。这是产品经理视角的「上帝视角」。

## 核心思想：产品矩阵是一个「价值漏斗」⭐

CEX 的产品不是随便堆的，而是一条精心设计的、把用户**逐步推向「更高频、更高费、更深绑定」**的漏斗。

```
                  用户旅程（也是交易所的"收割"路径）
  ┌────────────────────────────────────────────────────────────┐
  │                                                              │
  │  一键买币 ──▶ 现货交易 ──▶ 理财沉淀 ──▶ 合约 ──▶ 平台币/生态     │
  │  (拉新)      (留存)       (锁资金)     (高利润)   (深绑定)      │
  │                                                              │
  │  低门槛低风险 ───────────────────────────▶ 高门槛高风险高利润   │
  │  对交易所利润贡献：低 ──────────────────────────────▶ 高       │
  └────────────────────────────────────────────────────────────┘
```

每一层都解决一个商业目标：
1. **一键买币**：降低门槛，把小白拉进来。
2. **现货**：让用户学会交易，养成习惯。
3. **理财**：让资金沉淀下来不流失，同时成为放贷弹药。
4. **合约**：引导用户进入高频、高费率的衍生品（交易所利润核心）。
5. **平台币/生态**：用利益把用户深度绑定，提高离开成本。

## 用户分层运营

不同用户用不同的产品和界面伺候：

| 用户类型 | 特征 | 主推产品 | 界面 |
| --- | --- | --- | --- |
| 小白散户 | 不懂、怕复杂 | 一键买币、活期理财、跟单 | 简易模式 |
| 进阶散户 | 会看 K 线、想赚快钱 | 现货、低倍合约、Launchpad | 专业模式 |
| 高频玩家 | 量化、套利 | 高倍合约、API 交易、返佣 | API + 高级订单 |
| 机构/大户 | 大额、要服务 | OTC、托管、专属做市 | 机构专线 |

> 💡 **跟单交易（Copy Trading）**：近年的获客利器。让小白「一键复制」高手的操作。
> 表面是帮小白，实质是把小白引入高频合约（小白自己不会做合约，但敢跟单），极大提升交易量和手续费。

## 几个关键的产品设计套路

### 1. 默认引导向高利润产品
- 注册后引导你买币 → 买完币引导你「让闲置币赚收益」（理财）→ 不断推送合约活动。
- 默认选项、红点提示、活动弹窗都在往高利润产品导流。

### 2. 简易模式藏起复杂性、放大「赚钱想象」
- 把合约包装成「涨跌预测」「秒合约」，弱化「这是高杠杆赌博」的认知。
- ⚠️ 这是被监管诟病的「诱导性设计」。

### 3. 游戏化与即时反馈
- 盈亏实时跳动、爆仓动画、交易大赛排行榜、签到领空投。
- 用游戏化机制刺激多巴胺，提升交易频率（频率 = 手续费）。

### 4. 手续费的「温水煮青蛙」
- 单笔手续费看着很小（0.1%），用户不敏感。
- 但高频交易 + 高杠杆（杠杆放大了名义交易额）下，累积手续费极其可观。

### 5. 产品间的「交叉导流」
- 理财的币可以一键转去做合约保证金。
- 打新（Launchpad）要持有平台币。
- 处处设计「跳转到另一个产品」的钩子，提升单用户产品使用数（粘性指标）。

## 增长指标（产品经理盯什么）

| 指标 | 含义 |
| --- | --- |
| **DAU/MAU** | 日活/月活用户 |
| **交易量** | 核心，直接关联手续费收入 |
| **合约渗透率** | 多少用户用了合约（利润关键） |
| **资产留存(AUM)** | 用户存了多少资产在平台 |
| **单用户产品数** | 用户用了几个产品（粘性） |
| **提现率** | 提现占比（太高是流失/信任信号） |

## 一个诚实的视角 ⚠️

> 这个漏斗设计在商业上非常成功，但要清醒认识到：
> **交易所的利益最大化方向（高频、高杠杆交易），和散户的利益（少亏钱）是冲突的。**
> 交易所赚的是手续费（旱涝保收），用户在零和/负和的合约市场里整体是亏的。
> 理解产品设计哲学，不是为了膜拜它的精巧，而是为了**看清自己在漏斗里的位置，避免被收割。**

## 小结

- CEX 产品矩阵是一个把用户推向「高频高费高绑定」的价值漏斗。
- 通过用户分层、默认引导、游戏化、交叉导流等套路最大化交易量和留存。
- 跟单交易是把小白引入合约的典型获客设计。
- 关键认知：交易所的利益方向与散户利益存在结构性冲突。
