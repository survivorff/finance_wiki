# 3.3 订单类型

> 深度：🟡 进阶
> 订单类型和 [CEX](../../crypto/cex/02-产品体系/2.1-现货交易.md) 高度重叠。已读过 CEX 的可快速过，重点看股市特有的。

## 基础订单类型（同 CEX）

| 类型 | 含义 | 适用 |
| --- | --- | --- |
| **市价单 Market** | 按当前最优价立即成交 | 急于成交 |
| **限价单 Limit** | 指定价格，不满足就挂着等 | 想要理想价格 |
| **止损单 Stop** | 价格触发后转市价单 | 止损/突破 |
| **止损限价 Stop-Limit** | 触发后转限价单 | 止损但控制成交价 |

（详细机制见 [CEX 2.1](../../crypto/cex/02-产品体系/2.1-现货交易.md)，完全一致）

## 时效与条件（同 CEX）

- **当日有效（Day）**：当天收盘未成交则取消。
- **GTC（Good-Till-Cancelled）**：一直有效直到成交或手动取消。
- **IOC / FOK**：立即成交剩余取消 / 全部成交否则取消。
- **AON（All-Or-None）**：全部成交否则不成交（不一定立即）。

## 股市特有的订单类型 ⭐

### 1. 市价/限价 + 开盘/收盘（MOO / MOC / LOO / LOC）
```
MOO(Market-On-Open): 按开盘集合竞价价成交
MOC(Market-On-Close): 按收盘集合竞价价成交 ← 极常用!
  → 被动基金/ETF 调仓常用 MOC, 确保按官方收盘价成交
```
- 💡 这对应了 [3.2 集合竞价](./3.2-竞价机制.md) 的开收盘机制——CEX 没有「收盘」所以没有这类订单。

### 2. 隐藏单 / 冰山单（Hidden / Iceberg）
```
冰山单: 大单只显示一小部分在盘口, 成交一部分再露出一部分
  → 隐藏真实交易意图, 避免被市场看到大单而提前反应
```
- 同 [CEX 冰山订单](../../crypto/cex/08-深水区/8.1-撮合引擎实现细节.md)。机构大单常用。

### 3. 钉住/盯市单（Pegged Order）
```
报价自动"钉住"某个基准(如最优买价、中间价), 随行情自动调整
→ 做市商和算法常用
```

### 4. VWAP / TWAP 算法单（机构必备）⭐
```
不是单一订单, 而是"算法"把大单拆成很多小单, 在一段时间内执行:
  TWAP(时间加权): 均匀时间分布下单 → 平均时间价
  VWAP(成交量加权): 按市场成交量分布下单 → 贴近市场均价
目的: 大单不冲击市场、不暴露意图、拿到好均价
```
- 💡 **VWAP/TWAP 是机构交易的标配**。对照 [DEX 用 TWAP 做预言机](../../crypto/dex/03-技术架构/3.4-预言机与价格.md) 和 [DEX 聚合器拆单](../../crypto/dex/02-产品与机制/2.4-聚合器与意图.md)——「拆大单、防冲击、防被发现」是所有市场大额交易的共同智慧。

## 为什么订单类型这么多 💡

```
核心矛盾(所有市场通用):
  你想成交, 但又不想:
  - 暴露意图(被别人抢跑, 同 MEV/抢跑)
  - 冲击价格(大单推动价格不利于自己, 同 DEX 滑点)
  - 成交在坏价格
→ 各种订单类型/算法单, 都是在"成交确定性 vs 价格 vs 隐蔽性"间权衡
```

> 💡 这和 [CEX 订单类型](../../crypto/cex/02-产品体系/2.1-现货交易.md)、[DEX 防 MEV](../../crypto/dex/05-市场与博弈/5.2-MEV详解.md) 是同一组矛盾的不同解法。
> 大额交易者在哪个市场都要解决「如何不暴露、不冲击、拿好价」——这是市场微观结构的永恒主题。

## 小结

- 基础订单（市价/限价/止损）和时效（Day/GTC/IOC/FOK）与 CEX 一致。
- 股市特有：MOO/MOC（开收盘集合竞价单，机构调仓常用）、冰山单、钉住单。
- VWAP/TWAP 算法单是机构标配，把大单拆散执行，防冲击防暴露。
- 所有订单类型都在解决「成交确定性 vs 价格 vs 隐蔽性」的权衡——三个市场共通。
