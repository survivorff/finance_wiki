# 4.1 清算与交收概念

> 深度：🔴 硬核 ⭐
> 「成交」不等于「到手」。这一篇把交易、清算、交收三个概念彻底讲清，是理解股市基础设施的钥匙。

## 三个概念：交易、清算、交收 ⭐

```
1. 交易(Trade / Execution): 买卖双方在交易所"达成成交协议"
   → 秒级完成, 但此时钱和股票都没动! 只是"约定好了"

2. 清算(Clearing): 确认这笔交易、计算双方该付多少钱/该交多少股
   → 由清算所处理, 涉及轧差(netting)、对手方担保(CCP)

3. 交收(Settlement): 真正的"钱货两清" — 股票过户给买方, 钱划给卖方
   → 这才是交易真正"完成", 通常发生在成交后 1-2 天(T+N)
```

> ⭐ **核心认知**：你在 App 上看到「成交」的那一刻，**钱和股票还没真正交换**。
> 真正的交收发生在之后。中间这段时间，靠清算所担保「双方一定会履约」。

## T+N：交收周期 ⭐

```
T = 交易日(Trade date)
T+1 = 成交后第1个工作日交收
T+2 = 成交后第2个工作日交收

历史演进: T+5 → T+3 → T+2 → T+1 (美股 2024 年起 T+1)
中国 A 股: 股票 T+1(当天买当天不能卖), 资金 T+0/T+1
趋势: 不断缩短, 终极目标 T+0(实时交收)
```

> 💡 **为什么不立刻交收（T+0）？历史原因 + 风险管理**：
> - 历史：早期纸质股票，需要时间physically 转移和核对。
> - 现在：批量轧差（netting）能大幅减少实际要转移的钱货量（见下），需要一个处理窗口。
> - 风险缓冲：给清算所时间处理违约、核对。

## 轧差（Netting）⭐ 为什么交收能高效

```
不轧差: 券商 A 当天和别人做了 1000 笔交易, 每笔都单独交收 → 海量转账
轧差: 把 A 当天所有买卖"净额计算":
  A 总共买入 5000 股 AAPL, 卖出 4800 股 AAPL → 净买入 200 股
  → 只需交收"净额 200 股", 而不是 9800 股的总额!
→ 大幅减少实际需要转移的钱和股票(可能减少 90%+)
```

- 💡 这是清算所的核心价值之一：**用轧差把海量交易压缩成少量净额交收**，极大提升效率、降低风险。
- 对照 [CEX 内部记账](../../crypto/cex/03-技术架构/3.4-账户与清结算.md)：CEX 也是「内部轧差」（用户间交易只改余额），但 CEX 是自己一家做，股市是独立清算所做。

## 交收的两个方向：DVP ⭐

```
DVP(Delivery versus Payment, 券款对付):
  "交股票"和"付钱"必须同时发生, 一手交钱一手交货
  → 防止"给了钱没收到股票"或"给了股票没收到钱"的风险
  → 这是交收的黄金原则
```

- 💡 DVP 是交收安全的核心——**原子性的钱货两清**。
- 对照 [DEX 的原子结算](../../crypto/dex/01-行业全景/1.4-DeFi生态位.md)：DEX 用区块链原子性天然实现了「钱货两清不可分割」（一笔交易要么全成要么回滚），某种意义上是 DVP 的链上极致版本（且实时）！

## 整个流程串起来

```
T 日:    你下单买 100 股 → 交易所撮合成交(秒级)
         → 此时只是"约定", 钱和股票未动
T 日盘后: 清算所清算, 轧差计算各券商净额, CCP 介入担保(见 4.2)
T+1 日:  交收 — DVP 钱货两清:
         你的钱划走 → 股票登记到你名下(在 CSD, 券商代持, 见 4.3)
         交易真正"完成"
```

## 交收失败（Settlement Fail）💡

```
如果到交收日, 卖方交不出股票(如裸卖空) 或 买方付不出钱:
  → 交收失败
  → CCP 介入处理(强制买入/罚款/动用保证金, 见 4.2)
→ 这就是为什么需要 CCP 担保和保证金制度
```

## 小结

- 交易（达成协议，秒级）≠ 清算（计算轧差、担保）≠ 交收（钱货两清，T+1/T+2）。
- 成交那一刻钱和股票还没真正交换，靠清算所担保后续履约。
- 轧差把海量交易压缩成少量净额，是交收高效的关键。
- DVP（券款对付）是交收黄金原则——钱货同时交换，DEX 用链上原子性实现了实时版。
- 交收周期不断缩短（T+1），终极目标是实时（T+0）。
