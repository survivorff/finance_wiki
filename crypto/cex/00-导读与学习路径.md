# 00 · 导读与学习方法

> 在一头扎进细节前，先花 5 分钟理解「这个知识库怎么用」，能让后面的学习效率翻倍。

## 这个知识库为谁而写

- **小白**：从没接触过交易所，想搞懂「这玩意儿到底是什么、怎么运转、有什么坑」。
- **想入行的人**：产品、运营、BD、市场岗位，想了解交易所内部怎么做生意。
- **工程师**：想理解撮合、钱包、清算、风控这些系统怎么设计。
- **从业者**：想看到平时不写在公开文档里的运营套路、灰色地带、博弈内幕。

为此，每篇文章都会标注**深度等级**，并尽量做到：先用大白话讲清「是什么」，再深入「怎么做、为什么、有什么内幕」。

## 七大维度怎么来的

理解一家交易所，本质要回答 7 个问题：

| 维度 | 回答的问题 | 类比一家公司 |
| --- | --- | --- |
| 01 行业全景 | 这是什么行业、谁在玩 | 行业研报 |
| 02 产品体系 | 它卖什么给用户 | 产品线 |
| 03 技术架构 | 产品靠什么系统实现 | 研发/工程 |
| 04 运营体系 | 怎么获客、怎么维持运转 | 运营/市场/BD |
| 05 市场与博弈 | 怎么赚钱、各方怎么斗 | 商业模式/竞争 |
| 06 风险与监管 | 会怎么死、被谁管 | 风控/法务 |
| 07 未来发展 | 往哪走 | 战略 |

这 7 个维度合起来，就是一家交易所的完整切片。

## 怎么读（三条路径）

见 [README 的三条学习路径](./README.md#-三条学习路径按你是谁选)。简单说：
- 只想看懂 → 走「小白路径」，跳过 03 技术。
- 想入行 → 走「业务路径」，重点 02 + 04 + 05。
- 写代码 → 走「工程路径」，重点 03。

## 阅读约定

- **🟢🟡🔴** 标注深度，按需取舍。
- **⭐** 标注「核心中的核心」，时间紧只读带星的。
- **💡 内幕 / 冷知识** 标注行业里「不写在官方文档」的东西。
- **⚠️ 误区** 标注小白常见的误解。
- 涉及具体数字时，追求**量级正确**而非实时精确（市场变化太快）。

## 一个重要的心态

加密交易所行业**信息极度不对称**，官方话术、营销叙事、真实运作之间经常有巨大鸿沟。这个知识库会尽量戳破话术、还原真实逻辑。但也请保持独立判断——**任何关于「稳赚」「内部消息」的说法都值得怀疑**。学习的目的是看懂游戏规则，不是教你赌博。
